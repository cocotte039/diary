# Aesthete — 体験・認知負荷・静けさ

## 視点

「静けさ原則」（AGENTS.md #17）に立脚。本件は**両方とも「静けさを強化する」方向の変更**であり、アプリの世界観と合致する。UX の微調整ポイントを明確化する。

## 分析

### 変更1: 設定リンク削除 — 静けさの改善

EditorPage ヘッダーは現在 3 クラスタ: `[本棚] [< n / total >] [📅 設定]`。

**問題**:
- 日記を「書く」という一点集中の画面に、導線が多すぎる。特に「設定」は書く行為と無関係。
- 右上の 2 要素（日付ボタン + 設定リンク）が視覚的に混雑し、ヘッダー右端の情報密度が高い。
- 本棚メニューに既に設定があり、**二重導線**になっている（DRY 違反）。

**削除後の情報構造**:
```
[本棚] [< 3 / 60 >] [📅]
```
- 左: 文脈を捨てる入口（本棚）
- 中央: 現在位置の把握
- 右: 本文に関する操作（日付挿入）

ヘッダーが「ナビゲーション」と「編集アクション」で意味的に分離され、認知負荷が下がる。**日付ボタンは本文編集のアクションなので残すことに強い整合性がある**。

**CSS への配慮**:
- `.headerRight` は `gap: 0.25rem` で子が並ぶ。子が 1 つになった瞬間 `gap` は無効（描画に影響なし）。
- `justify-self: end` で日付ボタンが右端にピタリと張り付く。現在のレイアウトとの差分は「設定リンクの幅ぶんヘッダー右端が空く」だけ。
- Grid `1fr auto 1fr` の中央ピン留めは変わらない。ページ番号の視覚的中央位置が揺れない。

**美しさの観点**: ヘッダーの対称性が **本棚リンク**（テキスト）と **日付ボタン**（アイコン）という異種の 2 要素で崩れる懸念はあるが、左右とも 44px 相当の hit area を持ち、視覚重量は近い。許容範囲。

### 変更2: 戻るボタン → 本棚 — 場所感覚の整合

**現状の UX 問題**:
- 本棚 → 冊を開く → ページめくり → 戻るボタン で「ひとつ前のページ」（例: /book/v/2）に戻る
- ユーザーの心理モデル: 「戻る = 本棚（本を閉じる）」
- 実装の挙動: 「戻る = スタックを 1 段戻す」
- このズレは「Android の物理ボタンが知らない場所に連れていく」感覚を生み、**場所感覚の不安**を生む

**改善後の UX**:
- 本棚 → 冊を開く → ページめくり → 戻るボタン → **本棚**
- ユーザーの「本を閉じる」心理モデルと一致
- 静けさ原則: 「明示的な閉じる動作（本棚リンクのタップ）」と「暗黙的な閉じる動作（戻るボタン）」が同じ結果になる = **動線の冗長性が正しく縮退している**

**認知負荷**:
- ページめくり自体は履歴に蓄積されないほうが自然（日記は 1 冊の連続した時間軸であり、ページめくりは「本を読んでいる途中の位置」であって別ページへの「遷移」ではない）
- 戻るボタンで「本を閉じる」は、紙の日記を閉じる動作のメタファーとして直感的

**注意点**:
- 🟡 replace 時の一瞬のフラッシュ（前ページ表示 → `/`）: 180ms 以内なら視認されないが、Skeptic C1/C2 の通り保険が必要
- 静けさ原則上 **トースト・確認ダイアログは出さない**。戻るボタン = 即 本棚、が正しい

### モーダル系との協調（スコープ外確認）

- 本棚カレンダーモーダル（全画面モーダル）の戻るボタン挙動は非目標
- 今回の変更は EditorPage ローカルに閉じており、BookshelfPage の挙動に影響しない
- **将来の課題として**: BookshelfPage でカレンダーモーダル開いている時に戻るボタン → モーダル閉じるのが筋。ただし今回のスコープ外。plan.md の「未解決事項」に記載することを推奨

### ヘッダーの「静けさ」完成度

変更後のヘッダーを M8-UX / M9-M1 / M10-M2 の積み重ねと並べて見ると:

| 要素 | 目的 | 静けさ |
|---|---|---|
| 本棚リンク | 閉じる | テキスト、控えめ |
| ページ番号 | 現在位置 | opacity 0.5、数字のみ |
| 左右ボタン ‹› | ページめくり | opacity 0.3、枠なし |
| 日付ボタン 📅 | 日付挿入 | opacity 0.3、アイコン |
| プログレスバー | 残量 | 3px、モノクロ、10分割 |

→ すべて「主張しない」設計。設定リンク削除でこの静けさが完成度を増す。

## 推奨

1. **設定リンクを削除する**: 二重導線の解消、ヘッダーの静けさ強化、編集行為と無関係な要素を除去
2. **戻るボタンで本棚に戻す**: 場所感覚の整合、「本を閉じる」メタファー
3. **CSS 調整不要**: `.headerRight` の `gap` と grid 中央ピン留めは既に子要素数に堅牢

## 体験レビュー時の目視チェック項目

- [ ] ヘッダー右端が間延びして見えないか（日付ボタン 44x44 のままで OK）
- [ ] ページ番号が完全に画面中央に来るか（1fr auto 1fr で担保される）
- [ ] 戻るボタン押下時の視覚的フラッシュ（replace 前の一瞬）が知覚されないか
- [ ] ページめくり直後（フェード 180ms 中）の戻るで違和感がないか
- [ ] 本棚メニューの「設定」項目が見つけやすいか（導線が一本化される効果）
