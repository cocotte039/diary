# Aesthete 視点 — 仕様改善5件（2026-04-15）

## 視点宣言

視覚的美しさ・UX・一貫性・認知負荷・注意管理。
「静けさ原則」（color数最小・通知なし・200ms以内）に忠実に、「ノートらしさ」を損なわない。

---

## 1. 罫線均一化（①）

### 評価: 🔵 視覚的にむしろ静かになる

15/30/45 行目の強調は「紙の 1/4 区切り」という親切心だが、ユーザー視点で **不要な視覚ノイズ**だった。
`--color-rule` (rgba 0.08) のみの単一周期は、より「綺麗な原稿用紙」感を出す。

### Aesthete からの追加提案

- なし。合意設計のとおり、レイヤー削除のみで十分静かになる
- 一切の追加装飾を入れない（テクスチャ、影、グラデなど）

### 視覚整合性

- ヘッダー直下のプログレスバー (3px, 10 分割 tick) と罫線が干渉しないか確認
  - プログレスバーは `--color-rule` ベース・opacity 0.15 / 0.3 / 0.5 で構成
  - 罫線は opacity 0.08 で薄い
  - **干渉なし**: プログレスバーの方が濃く、視線階層として上位
- VolumeCard の border 1px solid `--color-rule` も同色だが用途が違うので問題なし

---

## 2. 日付挿入時のスクロール保持（②）

### 評価: 🔵 純粋な体験改善

現状の「日付挿入 → ページ最上部へ強制スクロール」は、**書く動作の連続性を断つ最悪の挙動**。修正で「目立たない」UX に戻る。

### Aesthete 観点での要件

- アニメーションで scrollTop を補間しない（即時復元）。なぜなら「本来動かないべきもの」だから
- focus による視覚カーソル移動は維持（カーソルは挿入位置に立つ）
- ユーザーには「日付が挿入され、カーソルが進んだ」だけが認識されるのが理想

### 静けさへの貢献

- スクロール戻りという「画面の暴れ」が消える
- 連続性が保たれることで、書くフロー（執筆中の集中）が破壊されない

---

## 3. 本棚並び順修正（③）

### 評価: 🔵 視覚的安定性の回復

並び順が不定だと「私のノートはどこに行った？」という認知負荷が発生する。これは **ユーザー報告の通り障害**。

### Aesthete 提案

- ordinal 降順は「最新を上に」というメンタルモデルと一致（時系列表示のスタンダード）
- 視覚上「左上が最新（書きかけ）、右下に向かって過去」のグリッドが完成する
- カードの `cardActive` 演出（4px translate + box-shadow）が **左上に常駐**することで、視線ガイドが安定する

---

## 4. 新ノート作成 UI のメニュー統合（④）

### 評価: 🔵 静けさへの大きな貢献

破線カード (`.newCard`, opacity 0.5) は、本棚グリッドに「常に薄く存在する点線」として **視覚的なザラつき**を生んでいた。これを削除することで、本棚が「ノートだけが並ぶ静かな書架」になる。

### Aesthete 観点でのメニュー設計

#### 4.1 ハンバーガーアイコン
- 3 本線、stroke 1.5px、opacity 0.5（既存ヘッダーリンク 0.3 と比べ少し強め）
- 理由: 機能的 affordance を示す要素。0.3 だと「あって無いような」状態でユーザーが見落とす
- ボタン領域 44x44（合意）

#### 4.2 ドロップダウンの視覚スタイル
- 背景: `--color-bg`（同色だが border で分離）
- border: 1px solid `--color-rule`
- border-radius: 4px（カードと同じ角丸）
- padding: 0.25rem 0
- 影: なし（静けさ）

#### 4.3 メニュー項目
- font-family: `--font-family-ui`（ヘッダーリンクと統一、Klee One ではない）
- font-size: 0.875rem
- padding: 0.75rem 1rem（タップ領域 44px 確保）
- opacity: 0.75 → hover/focus で 1.0
- 区切り線なし（静けさ）

#### 4.4 メニュー項目の順序
合意: 「新しいノート / カレンダー / 設定」
これは **使用頻度の昇順 → 降順** にするのが UX 原則だが、要件④で「新ノートは使用頻度低い」と明記。
矛盾するが、**機能的グルーピング**で解釈すると:
- 新しいノート（作成系）
- カレンダー（参照系）
- 設定（補助系）

の順は **論理的に正しい**。採用。

## 5. カレンダー UI のメニュー＋モーダル化（⑤）

### 評価: 🟡 設計判断が複数あり

合意: 全画面モーダル overlay 表示。
Aesthete としては **複数の表示方法**の検討が必要。

### 検討した案

#### 案A: 中央に小モーダル（推奨・採用）
- overlay (rgba 0.5) + 中央 panel (max-width 360px)
- 既存 Calendar の見た目をそのまま流用
- 静けさ: ◎（既存スタイル流用、新規装飾なし）
- 実装: ◎（Calendar 改変なし）

#### 案B: ヘッダーの直下に展開（dropdown 風）
- メニューと同じ位置に大きく展開
- 静けさ: △（メニューとカレンダーが似た位置）
- 体験: × （カレンダーは画面の主役、隅に追いやられる感）

#### 案C: フルスクリーン
- 100dvh 全面でカレンダー表示
- 静けさ: ○
- 体験: △ （閉じる動作が曖昧、本棚に戻る感がない）

**採用: 案A**。

### Aesthete 観点でのモーダル詳細

#### 5.1 overlay
- background: rgba(0, 0, 0, 0.5)（black overlay）
  - **代替案**: rgba(28, 28, 32, 0.85)（`--color-bg` ベース）→ より「自分の世界の延長」感が出る
  - 推奨: `rgba(0, 0, 0, 0.6)` で背景の本棚を遠景化（モーダルへの視線集中を作る）

#### 5.2 panel
- background: `--color-bg`（不透明）
- border: 1px solid `--color-rule`
- border-radius: 4px（カードと統一）
- padding: 1rem
- max-width: 360px（カレンダーの自然サイズ）
- max-height: calc(100dvh - 4rem)
- 影: なし（静けさ）。border のみで panel を分離

#### 5.3 閉じるボタン
- 右上に × アイコン（`×` 文字 or SVG）
- 44x44 タップ領域
- opacity 0.5 → active 0.8
- font-size: 1.25rem

#### 5.4 fade-in
- opacity 0 → 1 を 200ms（合意「200ms トランジション」準拠）
- transform: scale(0.96) → 1 を **見送る**（静けさ。透明度のみで充分）

実装案:

```css
.calendarOverlay {
  animation: fadeIn 200ms ease;
}
@keyframes fadeIn { from { opacity: 0; } to { opacity: 1; } }
```

#### 5.5 認知負荷
- カレンダー本体は既存通り（前月/次月、日付セル、ドット）
- モーダル化により「本棚の延長」ではなく「日付ジャンプ専用画面」として隔離される → 集中しやすい
- 日付タップで navigate → 本棚 unmount → モーダルも消える（自然な閉じ動作）

## ヘッダーメニュー基盤の Aesthete 評価

### 認知負荷の比較

#### 旧UI
- 本棚下部に「カレンダーを開く」ボタン（小さく opacity 0.5）
- グリッド末尾に「＋ 新しいノート」破線カード
- ヘッダー右に「設定」リンク

→ **3 か所に機能が散在**、ユーザーは「どこにあるか」を毎回探す必要があった

#### 新UI
- ヘッダー右にハンバーガーアイコン 1 つ
- 開くと「新しいノート / カレンダー / 設定」が縦に並ぶ

→ **機能の場所が 1 箇所に集約**、認知負荷↓

### 視覚階層

```
ヘッダー領域
├ 左: 本棚（h1）
└ 右: ハンバーガーメニュー（44x44, opacity 0.5）
本体
├ ノートグリッド（VolumeCard のみ、新ノートカードなし）
└ （カレンダートグルなし）
```

ノイズが取り除かれ、本棚は **「書架」** としての美しさを獲得。

### ヘッダーの opacity 一貫性

- ハンバーガー: 0.5（機能的 affordance）
- 既存「設定」リンク（旧）: 0.3

新しい設定リンクは BookshelfMenu 内に統合されるので、ヘッダー右はハンバーガーのみ。**opacity 0.5 で統一**。

### EditorPage との整合

- EditorPage は対象外（合意非目標）
- EditorPage のヘッダーは pageCluster + 設定リンク + 日付アイコンで既に複雑
- BookshelfPage のシンプル化と対比的に、EditorPage は「書く時の道具一覧」として残る

## トレードオフと判断

### 1. メニュー開閉アニメーション
- 案A: 即時表示・即時消滅
- 案B: opacity 100ms fade
- **採用: 案A（即時）**。理由: メニューは小さく、fade は冗長。即時の方がレスポンス感がある

### 2. カレンダーモーダル fade
- 案A: 即時
- 案B: 200ms fade
- **採用: 案B（200ms fade）**。理由: モーダルは視覚的に大きく、突然出ると驚きを与える。fade は「優しさ」の演出

### 3. 閉じるボタンの位置
- 案A: 右上 ×
- 案B: 下部「閉じる」テキストボタン
- **採用: 案A**。理由: 標準パターン、認知コストゼロ

### 4. メニュー項目数
- 現状 3 項目で適切
- 将来増やすなら 5 項目までが視覚的限界（スクロール無しで全部見える）

## 結論

合意済み設計は静けさ原則とノートアプリの世界観に **完全に整合**している。
特に「破線カード削除」「カレンダートグルボタン削除」「メニュー集約」は、本棚画面を **「書架」**として再定義する強い変更。

Aesthete としての追加提案は以下の 2 点のみ:
1. カレンダーモーダルに 200ms fade-in を追加（合意「200ms 以内」を最大限活用）
2. ハンバーガーアイコンの opacity は 0.5（既存 0.3 より少し強め、機能を発見させる）
