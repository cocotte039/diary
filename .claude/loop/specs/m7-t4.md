# M7-T4: `insertDate` を EditorPage へ移植

## 目的
WritePage の `formatToday()` / `insertDate` ロジックを EditorPage に移植し、ヘッダー右端の日付アイコンボタンから呼び出す。最下部の丸ボタン日付 UI は廃止。

## 変更対象
- ファイル:
  - `/home/cocotte/dev/diary/src/features/editor/EditorPage.tsx`
  - `/home/cocotte/dev/diary/src/features/editor/EditorPage.module.css`
- 削除対象:
  - WritePage 最下部の `styles.dateButton`（WritePage 自体は M7 T7.5 で削除）
- 追加:
  - `insertDate()`: textareaRef.current の selectionStart にスタンプを挿入、selection を更新、state を反映、onChange 系の処理（T6.3 含む）と協調

## 依存する既存コード
- `/home/cocotte/dev/diary/src/features/write/WritePage.tsx` の `formatToday()` / `insertDate` 既存実装（参照）
- T7.3 `DateIcon`
- T6.3 の自動遷移（insert 後に splitAtLine30 で overflow が出るなら遷移発動）

## 実装概要（3-5行）
1. `WEEKDAYS_JA` と `formatToday()` を EditorPage 内または `src/lib/date.ts` に関数として切り出し。
2. `insertDate = useCallback(() => { ... })` 実装:
   - `const el = textareaRef.current; if (!el) return;`
   - `const stamp = formatToday();`
   - `const { selectionStart, selectionEnd, value } = el;`
   - `const next = value.slice(0, selectionStart) + stamp + value.slice(selectionEnd);`
   - `setText(next);`（state 更新）
   - 次 tick で `el.setSelectionRange(selectionStart + stamp.length, selectionStart + stamp.length)` & `el.focus()`
   - overflow チェック（T6.3 のロジックを再利用）
3. ヘッダー右端に `<button aria-label="今日の日付を挿入" className={styles.headerDateButton} onClick={insertDate}><DateIcon /></button>`。
4. WritePage 側の dateButton UI は touch しない（M7 T7.5 で WritePage ごと削除）。

## 受入条件チェックリスト
- [ ] 🔵 ヘッダー右のアイコンをタップすると、カーソル位置に `YYYY年M月D日(曜)\n` が挿入される
- [ ] 🔵 挿入後のカーソル位置はスタンプ末尾
- [ ] 🔵 挿入で 30 行超過する場合、T6.3 の自動次ページ遷移が発動する
- [ ] 🔵 textarea のスクロール位置が破綻しない
- [ ] 🔵 ヘッダーのアイコンボタン hit area が 44x44 確保されている

## Key Links（配線検証）
- `insertDate` は EditorPage のヘッダーボタン onClick 本番経路で呼ばれる。
- `DateIcon` は button 内の子要素として本番レンダリング。

## テスト方針
- ユニット (Vitest + Testing Library):
  - EditorPage で `button[aria-label="今日の日付を挿入"]` を click
  - Date をモック (`vi.setSystemTime`) して決定論的に `2026年4月14日(火)\n` が挿入される
  - 挿入前後の selectionStart が `pos + stamp.length` に移動
  - 30 行目末尾で insertDate → 次ページに overflow が移る
- 目視: 本日の日付フォーマットが正しいか、曜日の括弧が全角か。

## 注意事項
- `WEEKDAYS_JA` の出力は `(月)` のように全角丸括弧ではなく半角丸括弧。既存 WritePage の仕様踏襲。
- `setText → setSelectionRange` の順序は React 19 でも `requestAnimationFrame` / `queueMicrotask` による再配置が必要な場合あり。既存 WritePage の実装を参考にする。
- 日付ロジックを `src/lib/date.ts` に切り出すのは任意（最小差分なら EditorPage 内で可）。
