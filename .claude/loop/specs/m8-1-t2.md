# T8-1.2: EditorPage ヘッダーを .app-header に置換

## 変更対象
- `src/features/editor/EditorPage.tsx`
- `src/features/editor/EditorPage.module.css`

## 依存
- T8-1.1（global.css に .app-header が存在すること）

## 実装概要
EditorPage の `<header>` に `app-header` クラスを併用し、module.css の `.header` から共通プロパティ（position/top/left/right/height/padding/background-color/z-index/flex 系/font-family）を削除する。左右 padding は `max(1rem, ...)` に統一される。

## 差分イメージ

### EditorPage.tsx
```tsx
// before
<header className={styles.header}>
// after
<header className={`app-header ${styles.header}`}>
```

### EditorPage.module.css
```css
/* before: .header が position/height/padding 等を全て持つ */
.header {
  position: fixed;
  top: 0;
  left: 0;
  right: 0;
  height: calc(var(--header-height) + env(safe-area-inset-top, 0px));
  padding-top: env(safe-area-inset-top, 0px);
  padding-left: max(0.5rem, env(safe-area-inset-left));
  padding-right: max(0.5rem, env(safe-area-inset-right));
  display: flex;
  justify-content: space-between;
  align-items: center;
  z-index: 2;
  background-color: var(--color-bg);
  font-family: var(--font-family-ui);
}

/* after: 空にする（.app-header 側で賄えるため）か削除。
 * 残すなら副次プロパティのみ（現状なし）。 */
```

`styles.header` を使い続けるため、空のクラス定義は残す（他の CSS セレクタから参照されるケースを想定）。あるいは JSX 側で `className="app-header"` のみに切り替える（テストで参照されない前提を確認）。

## 受入条件
- 🔵 EditorPage のヘッダーが `app-header` クラスを持つ（テスト可能）
- 🔵 EditorPage ヘッダーの左右 padding が `max(1rem, ...)` 由来（視覚確認）
- 🔵 既存の EditorPage.test.tsx が全通過

## テスト方針
- 既存テストは className 直接 assert していないので大半は無影響。
- ヘッダー要素の className に `app-header` が含まれることを 1 ケース追加してもよい。

## 注意事項
- DateIcon 挿入ボタン (headerDateButton) と pageCluster は CSS Module の styles を継続使用
- ヘッダー内 `.app-header-link` は既存踏襲
