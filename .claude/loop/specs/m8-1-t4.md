# T8-1.4: SettingsPage ヘッダー固定 + 縦スクロール

## 変更対象
- `src/features/settings/SettingsPage.tsx`
- `src/features/settings/SettingsPage.module.css`

## 依存
- T8-1.1（.app-header クラスが global.css にあること）

## 実装概要
T8-1.3 と同じパターンを SettingsPage に適用する。

## 差分イメージ

### SettingsPage.tsx
```tsx
<header className={`app-header ${styles.header}`}>
```

### SettingsPage.module.css
- `.root` を `height:100dvh` + `overflow-y:auto` + `padding-top: calc(var(--header-height) + env(safe-area-inset-top, 0px) + 1rem)`
- `.header` の margin-bottom を削除、app-header と重複するプロパティを除去

## 受入条件
- 🔵 SettingsPage `<header>` に `app-header` クラスが含まれる
- 🔵 `.root` に `overflow-y:auto` が設定される
- 🔵 既存 SettingsPage.test.tsx が全通過
- 🔵 最初のセクションと header の間に空間ができる（padding-top 分）

## テスト方針
- 新規: ヘッダー要素の className に `app-header` が含まれることを確認する 1 ケース追加

## 注意事項
- 設定ページは長いので縦スクロール必須
- フォーム入力時の iOS Safari のスクロール挙動（input focus 時のズレ）は env(safe-area-inset-bottom) で吸収
