# M5-T1: ページ遷移 UI（左右ボタン）

## 目的
EditorPage に左右ページ送りボタンを追加し、隣接ページへの遷移時に autosave flush と `lastOpenedPage` 更新を行う。

## 変更対象
- ファイル:
  - `/home/cocotte/dev/diary/src/features/editor/EditorPage.tsx`
  - `/home/cocotte/dev/diary/src/features/editor/EditorPage.module.css`
- 追加要素:
  - `<button aria-label="前のページ">‹</button>` / `<button aria-label="次のページ">›</button>`（モノクロ、opacity 0.3、active 時 0.6）
  - 配置: ヘッダー直下、ページ番号の左右、または画面端固定（推奨: ページ番号と同行）
- 関数:
  - `goPage(delta: number)` を内部に追加

## 依存する既存コード
- `/home/cocotte/dev/diary/src/lib/db.ts` `updateVolumeLastOpenedPage`（T4.2）
- `/home/cocotte/dev/diary/src/features/editor/useEditorAutoSave.ts` の `flush`（T4.4）
- `/home/cocotte/dev/diary/src/lib/constants.ts` `PAGES_PER_VOLUME`
- `react-router-dom` `useNavigate`

## 実装概要（3-5行）
1. `goPage(delta)` で `next = current + delta`。`next < 1` または `next > PAGES_PER_VOLUME` で no-op。
2. `await flush()`（autosave 即時反映）。
3. `await updateVolumeLastOpenedPage(volumeId, next)`。
4. `navigate(\`/book/${volumeId}/${next}\`)`（`replace: false`、ブラウザ戻るを残す）。
5. ボタン disabled: `current === 1` で「前」、`current === PAGES_PER_VOLUME` で「次」（M6 のロックは別途）。

## 受入条件チェックリスト
- [ ] 🔵 「次へ」ボタンで `pageNumber + 1` のページに遷移する
- [ ] 🔵 「前へ」ボタンで `pageNumber - 1` のページに遷移する
- [ ] 🔵 1 ページ目で「前」ボタンが無効
- [ ] 🔵 50 ページ目で「次」ボタンが無効（M6 まで暫定）
- [ ] 🔵 遷移前に `flush` が呼ばれデータが保存される
- [ ] 🔵 遷移時に `updateVolumeLastOpenedPage` が呼ばれる

## Key Links（配線検証）
- `goPage` は EditorPage のボタン onClick から呼ばれる本番経路。
- T5.5 の PageUp/PageDown でも同じ `goPage` を使うこと（コード重複回避＆配線統一）。

## テスト方針
- ユニット (Vitest + Testing Library):
  - DB に volume + page #1, #2 を仕込み、`/book/{id}/1` でレンダ → 「次へ」クリック → `flush` 解決後に navigate が `/book/{id}/2`
  - `updateVolumeLastOpenedPage` が呼ばれ DB の volume の `lastOpenedPage === 2`
  - 1 ページ目で「前」ボタンが `disabled`
  - flush の呼び出し順序: 「保存 → DB更新 → navigate」を spy で確認
- 目視: ボタンタップで滑らかに遷移する（フェードは T5.2）。

## 注意事項
- 最終ページの「次」挙動は M6 T6.3/T6.4 で確定（自動次ページ作成 or ロック）。M5 では disabled で良い。
- `replace: false` にしてブラウザの戻る/進むでもページ移動できるようにする（既存 ReaderPage は replace: true だが、編集ページは履歴を残した方が UX が良い）。
