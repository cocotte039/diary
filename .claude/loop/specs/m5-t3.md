# M5-T3: 左右スワイプ対応（textarea 競合回避）

## 目的
EditorPage で左右スワイプによるページめくりを可能にする。textarea のカーソル移動・選択操作を妨げないため、まずは A 案（textarea 以外の余白領域のみ反応）で実装する。

## 変更対象
- ファイル:
  - `/home/cocotte/dev/diary/src/features/editor/EditorPage.tsx`
  - `/home/cocotte/dev/diary/src/features/editor/EditorPage.module.css`
- 追加要素:
  - root 要素の `onTouchStart` / `onTouchEnd` ハンドラ
  - スワイプ判定領域: ヘッダー下余白 / textarea 上下マージン / ページ番号周辺（textarea 自身ではない領域）
- 関数:
  - `onSwipeStart(e)` `onSwipeEnd(e)` を追加。`textarea` 内発生のイベントは無視（`e.target` が textarea なら return）

## 依存する既存コード
- `/home/cocotte/dev/diary/src/lib/constants.ts` `SWIPE_THRESHOLD_PX = 50`
- `/home/cocotte/dev/diary/src/features/reader/ReaderPage.tsx`（移植元: `touchStartX` ref パターン）
- T5.1 `goPage(delta)`

## 実装概要（3-5行）
1. `touchStartX = useRef<number | null>(null)`、`touchStartY = useRef<number | null>(null)`。
2. `onTouchStart`: `e.target` が textarea 要素なら無視。それ以外で start 座標を ref に格納。
3. `onTouchEnd`: 同じく textarea 由来は無視。`dx = endX - startX`, `dy = endY - startY`。`Math.abs(dx) >= SWIPE_THRESHOLD_PX && Math.abs(dx) > Math.abs(dy)` で水平スワイプと判定。
4. dx<0 → `goPage(1)`、dx>0 → `goPage(-1)`。
5. root の `<div onTouchStart={...} onTouchEnd={...}>` で全領域捕捉、textarea 内は上記 target チェックで除外。

## テスト方針
- ユニット (Vitest + Testing Library):
  - root 要素 (data-testid="editor-root") に対し `fireEvent.touchStart` `fireEvent.touchEnd` で 60px 左移動 → `goPage(1)` 相当の挙動（navigate が次ページに）
