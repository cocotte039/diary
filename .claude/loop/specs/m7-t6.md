# M7-T6: ヘッダー視覚統一

## 目的
EditorPage / BookshelfPage / SettingsPage の各ヘッダー UI（リンクテキスト、アイコン）を同じフォント・opacity・ベースラインで揃え、ページ間の見え方を一貫させる。

## 変更対象
- ファイル:
  - `/home/cocotte/dev/diary/src/features/editor/EditorPage.module.css`
  - `/home/cocotte/dev/diary/src/features/bookshelf/BookshelfPage.module.css`
  - `/home/cocotte/dev/diary/src/features/settings/SettingsPage.module.css`
  - 必要なら `/home/cocotte/dev/diary/src/styles/global.css` に共通 `.app-header-link` クラス追加
- 変更内容:
  - ヘッダーのリンク・ボタン: `font-size: 0.75rem` / `opacity: 0.3` / `:active { opacity: 0.6 }`
  - アイコン (T7.3 DateIcon) とテキストのベースライン揃え（`display: inline-flex; align-items: center; gap: 0.25rem;`）
  - 色は `color: var(--color-ink)` 統一

## 依存する既存コード
- 既存 WritePage 側のフォントサイズ値 `0.75rem`
- T7.3 `DateIcon`（stroke currentColor なので color 連動）
- `global.css` トークン `--color-ink`

## 実装概要（3-5行）
1. `global.css` に `.app-header-link { font-size: 0.75rem; color: var(--color-ink); opacity: 0.3; transition: opacity 120ms ease; } .app-header-link:active { opacity: 0.6; }` を追加（`:global(...)` 経由で CSS Module 内でも使えるよう設計）。
2. EditorPage / BookshelfPage / SettingsPage の header JSX で Link/button に `className="app-header-link"` を付与。
3. ベースライン揃えは `.app-header-row { display: flex; align-items: center; gap: 0.75rem; }` を使用。
4. ページ固有の独自 opacity/font-size 指定があれば `.app-header-link` に統合するよう重複削除。

## 受入条件チェックリスト
- [ ] 🟡 EditorPage / BookshelfPage / SettingsPage でヘッダーリンクのフォントサイズが 0.75rem 統一 — 推測根拠: WritePage 既存値
- [ ] 🟡 ヘッダーリンクの opacity が 0.3、タップ時 0.6
- [ ] 🔵 DateIcon とテキストのベースラインが揃う
- [ ] 🔵 ページ間でヘッダーの見え方が一貫する（目視）

## Key Links（配線検証）
- `.app-header-link` クラスは各ページヘッダーの本番レンダリング要素に付与される。
- 単なる CSS 追加なので runtime 呼び出しは発生しないが、3 ページすべてで使われていることで配線を保証。

## テスト方針
- ユニット (Vitest + Testing Library):
  - 各ページをレンダし、ヘッダーリンクの `className` に `app-header-link` が含まれる
  - computed style でフォントサイズが `12px` (0.75rem)
- 目視: 3 ページを行き来してヘッダーの文字サイズ・薄さが揃っているか。

- BookshelfPage の既存 `.title` は `h1` の大きさ指定があるため、リンク統一とは別建てで扱う。
- SettingsPage の「戻る」リンクなど既存 UI テキストは現行ラベルを維持（T4.6 で「本棚」へ変更済み）。
