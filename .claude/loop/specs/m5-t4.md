# M5-T4: カーソル復元のページ単位スコープ化

## 目的
`useCursorRestore` が使うカーソル位置キーを volumeId + pageNumber 単位にスコープし、ページ A と B でカーソル位置が干渉しないようにする。

## 変更対象
- ファイル:
  - `/home/cocotte/dev/diary/src/hooks/useCursorRestore.ts`（または EditorPage 用フック新規）
  - `/home/cocotte/dev/diary/src/lib/constants.ts`（必要なら LS_CURSOR_KEY の扱いを prefix 化）
- 関数:
  - シグネチャ変更案: `useCursorRestore(ref, text, ready, scopeKey: string)` — `scopeKey` を localStorage key の後ろに付与
  - もしくは `EditorPage` 専用の `useEditorCursor(ref, text, ready, volumeId, pageNumber)` を新規作成し、既存 `useCursorRestore` は WritePage 用に残す（M7 で掃除）

## 依存する既存コード
- `/home/cocotte/dev/diary/src/hooks/useCursorRestore.ts` 現行実装
- `/home/cocotte/dev/diary/src/lib/constants.ts` `LS_CURSOR_KEY`

## 実装概要（3-5行）
1. EditorPage 用に新規 `useEditorCursor` を作成（既存フックに影響を与えない安全策）。
2. localStorage キー: `${LS_CURSOR_KEY}:${volumeId}:${pageNumber}`。
3. 取得・保存・復元ロジックは既存 `useCursorRestore` を踏襲。`restoredRef` を volumeId/pageNumber 依存で再評価するため `useEffect` の依存配列に両方追加し、ページ切替時に再復元。
4. `onSelectionChange(pos)` をコンポーネントから呼ぶインターフェイスは踏襲。

- [ ] 🔵 ページ切替後、textarea にフォーカスが戻りカーソルが復元される

- [ ] 🟡 従来の `LS_CURSOR_KEY` 単独キーが残っていても無視される（互換失敗しても編集できる）

## Key Links（配線検証）
- `useEditorCursor` は `EditorPage.tsx` の `useRef<HTMLTextAreaElement>` に対して直接呼ばれる本番経路。
- `onSelectionChange` は textarea の `onSelect` / `onChange` ハンドラから呼ばれる。

## テスト方針
- ユニット (Vitest + Testing Library):
  - mount `<EditorPage>` at `/book/v1/1`、textarea で selection を 10 に設定 → debounce 待ち → localStorage に `note-cursor-position:v1:1 = 10`
  - 同じ EditorPage を `/book/v1/2` に切替 → localStorage に `note-cursor-position:v1:2` が独立して書かれる
  - 再度 `/book/v1/1` に戻ると selection が 10 に復元
- 手動検証: カーソルを動かし、ページ移動→戻って同位置か確認。

## 注意事項
- 既存 `useCursorRestore` は WritePage 依存のため変更せず新規フック路線が低リスク。M7 T7.5 で WritePage 削除時に旧フックも除去。
- volumeId/pageNumber 変更時の再復元では `restoredRef.current` をリセットする必要あり（`useEffect` の最初で false に戻す、または scope をキー化する）。
- localStorage の空き容量（総書き込み量）は冊数×50ページで数百エントリ程度に留まり、5MB 上限に対して十分。
