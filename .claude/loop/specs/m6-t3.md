# M6-T3: 30行到達時の自動次ページ遷移

## 目的
EditorPage で入力が 30 行を超えたら、超過分を次ページ先頭に持ち越し自動遷移する。遷移後にカーソルを overflow 末尾に置く。

## 変更対象
- ファイル:
  - `/home/cocotte/dev/diary/src/features/editor/EditorPage.tsx`
- 動作:
  - `onChange` (composition 中でない場合) → `splitAtLine30(newValue)` → overflow 非空で自動遷移処理を走らせる
  - カーソル位置仕様: 次ページ先頭から `overflow.length`（オーバーフロー末尾）

## 依存する既存コード
- T6.1 `splitAtLine30`
- T6.2 IME ガード `isComposingRef`
- T4.2 `savePage`, `getPage`
- T4.4 `useEditorAutoSave` の `flush`（もしくは直接 `savePage`）
- T5.1 `goPage` / `navigate`、`updateVolumeLastOpenedPage`

## 実装概要（3-5行）
1. `onChange(e)` で `value = e.target.value`。`setText(value)` で state 反映。
2. composition 中でなく `pageNumber !== PAGES_PER_VOLUME`（50 は T6.4 でロック）の場合、`const { keep, overflow } = splitAtLine30(value)`。
3. overflow 非空なら:
   a. `await savePage(volumeId, current, keep)`（現ページを即時保存）
   b. 次ページの既存 content を `getPage(volumeId, current+1)` で取得（無ければ空）
   c. 新コンテンツ = `overflow + existingNextContent`（overflow を先頭に prepend）
   d. `await savePage(volumeId, current+1, newNextContent)`
   e. `await updateVolumeLastOpenedPage(volumeId, current+1)`
   f. `navigate('/book/{id}/{current+1}')`
4. 遷移後 useEffect（ページロード完了時）で textarea をフォーカス、`setSelectionRange(overflow.length, overflow.length)`。
5. `flush` タイマーは `lastSavedRef` 更新済みとして扱う（autosave が再発火しないよう注意）。

## 受入条件チェックリスト
- [ ] 🔵 30 行目末尾で改行すると、31 行目の内容が次ページ先頭に移動する
- [ ] 🔵 既に次ページに content がある場合、overflow が先頭に prepend され、既存 content はその後に残る
- [ ] 🔵 遷移後のカーソル位置が `overflow.length`（オーバーフロー末尾）
- [ ] 🔵 IME 変換中は遷移しない（T6.2）
- [ ] 🔵 50 ページ目では発動しない（T6.4 ロック）
- [ ] 🟡 複数改行で 2 ページ以上跨ぐ入力は 1 ページずつ連鎖処理される — 推測根拠: 巨大 overflow の安全側挙動

## Key Links（配線検証）
- `EditorPage.onChange` が本番配線。そこから `splitAtLine30` → `savePage` → `navigate` まで一気通貫。
- カーソル復元は T5.4 と連携せず、専用ロジック（次ページ到着後に `overflow.length` 位置へ移動）をここで持つ。

## テスト方針
- ユニット (Vitest + Testing Library + fake-indexeddb):
  - DB に volume + page1 (空) + page2 (空) を仕込む
  - page1 の textarea に 31 行のテキストを `fireEvent.change` → navigate が `/book/{id}/2` / page1 の content は 30 行に切り詰め / page2 の先頭に 31 行目の内容
  - page2 に既存 content "abc" がある場合、遷移後 page2 content が `{overflow}abc`
  - カーソル位置 = overflow.length（setSelectionRange のモック検証）
  - 50 ページ目での 31 行入力 → 遷移せず、T6.4 のロックに従う
- 目視: 実機で 30 行目末尾に改行を入れて自然に次ページへ移るか。

## 注意事項
- overflow が次ページ容量（30 行）を超える場合、次ページ側で再度 splitAtLine30 が連鎖発火する必要があるが、useEffect + onChange 経由で自動継続する（データロスを避けるため連鎖処理を許容）。
- `savePage` 2 連発の間に失敗した場合の復旧は将来課題。M6 ではベストエフォート。
- autosave の debounce タイマーと競合しないよう、自動遷移処理前に `flush` を呼ぶか、`lastSavedRef` を更新しておく。
