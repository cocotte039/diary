# T2.3: 読み返し画面

## 目的
過去の冊をページめくりで読み返す画面を実装する。書く画面と同じノート風スタイルで、静かな読書体験を提供する。

## 変更対象
- `diary/src/features/reader/ReaderPage.tsx` — 読み返し画面
- `diary/src/features/reader/ReaderPage.module.css` — 読み返しスタイル

## 詳細仕様

### ReaderPage.tsx
- URLパラメータ: `volumeId`, `pageNumber`
- IndexedDBから該当ページを取得
- 読み取り専用の div でテキスト表示（textareaではなく div + white-space: pre-wrap）
- 罫線・フォントは `notebook.css` を共有（書く画面と同じ見た目）

### ページ送り
- 左右スワイプでページ送り（素のtouchイベント）
  - `touchstart` で開始位置を記録
  - `touchend` で移動距離を計算
  - 閾値: 50px 以上の水平移動でページ送り
- 左スワイプ: 次のページ
- 右スワイプ: 前のページ
- 最初のページで右スワイプ → 何もしない
- 最後のページで左スワイプ → 何もしない
- ページ遷移アニメーション: CSS transition（opacity 200ms ease）

### UI要素
- 上部: 控えめなページインジケーター `x / 50`（opacity: 0.3）
- 上部左: 戻るボタン（← 本棚に戻る）
- その他のUI要素なし

## 注意事項
- div表示のため、textareaとは微妙にレンダリングが異なる可能性がある → padding/margin を揃える
- スワイプ操作がブラウザの戻る操作（iOS Safari）と衝突しないよう、touchイベントでpreventDefaultを適切に使う

## 完了サマリー

- 実変更:
  - `src/features/reader/ReaderPage.tsx` (読み取り専用 div、notebook-surface 共有、touch swipe)
  - `ReaderPage.module.css` (.surface と `.fading`)
- 判断:
  - スワイプ閾値 50px は SWIPE_THRESHOLD_PX 定数に切り出し。
  - ページ遷移は navigate(..., { replace: true }) で履歴を肥大化させない。
  - fade 180ms → navigate → fade戻し（計 ~200ms）で静かなめくり感。
- 注意:
  - iOS Safari の horizontal swipe は戻る操作と干渉する可能性がある。実機で確認し、必要なら `touch-action: pan-y` を付与する（別PC実機確認）。
