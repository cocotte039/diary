# T2.1: 冊の管理

## 目的
50ページに到達したら自動で新しい冊に切り替わり、ユーザーが任意タイミングで手動切替もできるようにする。冊の終わりが近いことを視覚的に伝える。

## 変更対象
- `diary/src/features/write/useWrite.ts` — 書く画面のカスタムフック（冊切替ロジック）
- `diary/src/lib/db.ts` — 冊切替のDB操作追加
- `diary/src/features/write/WritePage.tsx` — 手動切替UI追加
- `diary/src/features/write/WritePage.module.css` — 冊終わりの視覚表現

## 詳細仕様

### 自動切替ロジック（useWrite.ts）
1. テキスト保存時に論理行数を計算
2. 行数が `LINES_PER_PAGE * PAGES_PER_VOLUME`（1500行）に到達
3. IndexedDBトランザクションでアトミックに:
   - 旧Volumeの `status` を `'completed'` に更新
   - 新Volumeを `status: 'active'` で作成
   - 新Volume用のPage #1を作成
4. textareaをクリアし、新しい冊のPage #1から開始

### 手動切替
- 書く画面の控えめなメニュー内（画面上部隅のアイコン、opacity: 0.3）
- 「新しいノートに切り替える」ボタン
- 確認なし（実際のノートでは確認なく新しいノートを使い始める）
- 切替後、現在の冊の最後のページの空行はそのまま残す

### 🟡 冊の終わりの視覚表現
- 最後の5ページ（ページ46-50）: ページ区切り線の色味が微妙に暖色に変化
  - `rgba(255, 255, 255, 0.15)` → `rgba(255, 200, 150, 0.15)`
- 最後の2ページ（ページ49-50）: 余白がわずかに狭くなる（padding-right微減）
- 推測根拠: 「紙が残り少ない」物理的感覚のデジタル再現

### リカバリロジック
- 起動時に `status` チェック:
  - `active` な Volume が0個 → 新Volume作成
  - `active` な Volume が2個以上 → 最新以外を `completed` に修正
