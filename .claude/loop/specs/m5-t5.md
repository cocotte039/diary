# M5-T5: PageUp / PageDown キーでページ遷移

## 目的
キーボード操作でもページめくりできるよう、textarea で PageUp/PageDown を捕捉して `goPage` を発火させる。

## 変更対象
- ファイル:
  - `/home/cocotte/dev/diary/src/features/editor/EditorPage.tsx`
- 変更内容:
  - textarea の `onKeyDown` ハンドラを追加
  - `e.key === 'PageUp'` で `goPage(-1)`、`'PageDown'` で `goPage(1)`
  - 両者で `e.preventDefault()` を呼びデフォルトのスクロールを抑止

## 依存する既存コード
- T5.1 `goPage(delta)`
- React `KeyboardEvent<HTMLTextAreaElement>`

## 実装概要（3-5行）
1. `handleKeyDown(e: React.KeyboardEvent<HTMLTextAreaElement>)` を定義。
2. `if (e.key === 'PageUp') { e.preventDefault(); goPage(-1); }`、PageDown も対称。
3. textarea に `onKeyDown={handleKeyDown}` を配線。
4. composition 中（IME 入力中）は遷移しないよう `if (isComposingRef.current) return` を加える（T6.2 の ref を参照可。M6 完了前は省略可）。

## Key Links（配線検証）
- `handleKeyDown` は EditorPage の `<textarea onKeyDown={...}>` に直接配線される本番経路。
- 内部から T5.1 `goPage` を呼ぶことで flush + lastOpenedPage 更新も連動。

## テスト方針
- ユニット (Vitest + Testing Library):
  - textarea を取得 → `fireEvent.keyDown(textarea, { key: 'PageDown' })` → `goPage(1)` 相当の navigate 確認
  - 1 ページ目で PageUp → no-op
  - composition 中（`isComposingRef.current = true` 状態）で発火しない（T6.2 と協調確認）
- 目視: 物理キーボードで操作確認（モバイルでは未使用）。

## 注意事項
- モバイル（iOS/Android）では PageUp/PageDown キーは存在しないため、副次対応扱い。
- ブラウザによっては Fn キーを伴うため、ユーザー側の操作環境差は許容。
