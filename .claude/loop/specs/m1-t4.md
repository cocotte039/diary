# T1.4: テキストのローカル保存（IndexedDB）

## 目的
textareaのテキストをIndexedDBに自動保存し、ブラウザリロード後もデータが復元されるようにする。

## 変更対象

## 詳細仕様

### types/index.ts
```typescript
export interface Volume {
  id: string;           // crypto.randomUUID()
  createdAt: string;    // ISO8601
  status: 'active' | 'completed';
}

export interface Page {
  id: string;           // crypto.randomUUID()
  volumeId: string;     // FK -> Volume
  pageNumber: number;   // 1-50
  content: string;      // テキスト内容
  createdAt: string;    // ISO8601（初回書き込み時）
  updatedAt: string;    // ISO8601
  syncStatus: 'synced' | 'pending';  // Phase 3で使用、初期値 'pending'
}
```

### 保存ロジック（useAutoSave.ts）
1. textarea.value を `\n` で split して論理行の配列にする
2. 30行ごとにチャンクに分割: `lines.slice(i*30, (i+1)*30).join('\n')`
3. 各チャンクを対応する Page レコードの content として保存
4. 行数が増えて新しいページが必要になったら Page レコードを自動作成
5. debounce: 2秒（AUTOSAVE_DEBOUNCE_MS）

### 読み込みロジック
1. アクティブな Volume（status === 'active'）を取得
2. なければ Volume #1 と Page #1 を自動作成
3. Volume の全 Page を pageNumber 順に取得
4. content を `\n` で連結して textarea にセット

### navigator.storage.persist()
- 起動時に呼び出し（Eviction 対策）
- 結果はログ出力のみ（UIには表示しない）
