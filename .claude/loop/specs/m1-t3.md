# T1.3: ページの視覚的表現

## 目的
30行ごとのページ区切りを視覚的に表現し、日付入力補助ボタンを配置する。ユーザーが「ノートのページ」を感覚的に把握できるようにする。

## 変更対象
- `diary/src/features/write/WritePage.tsx` — 追記（ページ番号表示、日付ボタン）
- `diary/src/features/write/WritePage.module.css` — 追記（ページ区切り線、ボタンスタイル）
- `diary/src/lib/constants.ts` — 定数定義

## 詳細仕様

### constants.ts
```typescript
export const LINES_PER_PAGE = 30;
export const PAGES_PER_VOLUME = 50;
export const LINE_HEIGHT_EM = 1.8;
export const FONT_SIZE_PX = 16;
export const AUTOSAVE_DEBOUNCE_MS = 2000;
export const CURSOR_SAVE_DEBOUNCE_MS = 1000;
```

### ページ区切り線
- 30行ごとに罫線より少し太い（2px）、少し明るい色の区切り線
- CSS背景パターンに追加（repeating-linear-gradientのオーバーレイ）
- 区切り線の間隔: `LINES_PER_PAGE * LINE_HEIGHT_EM * FONT_SIZE_PX`

### ページ番号表示
- 画面右下隅に控えめな表示（`opacity: 0.3`）
- 現在のカーソル位置からページ番号を計算
- ユーティリティ関数: `getPageNumber(selectionStart: number, text: string): number`
  - textのselectionStartまでの`\n`の数をカウント → 30で割って+1

### 日付入力補助ボタン
- 画面下端、小さな丸ボタン（直径32px程度）
- `opacity: 0.25`、タップ時にのみ `opacity: 0.5` に変化
- タップで `「2026年4月13日\n」` をカーソル位置に挿入
- 日付フォーマット: `YYYY年M月D日`（和暦ではなく西暦）

## 受入条件（Must-haves）

### Truths
- 🔵 30行ごとにページ区切り線が表示される

- 🔵 画面右下にページ番号が表示される（opacity: 0.3）
- 🔵 日付ボタンをタップすると今日の日付がカーソル位置に挿入される

## テスト方針
- 30行以上のテキストを入力し、区切り線の位置が正しいことを目視確認
- 日付ボタンの動作確認（カーソル位置に挿入されること）
- ページ番号がカーソル移動に応じて更新されることを確認

## 依存関係

## 注意事項
- ページ区切り線は「目安」であり、厳密なページ制御ではない
- 折り返し行（word-wrap）はカウントしない仕様のため、画面幅によってページ区切りの位置と実際の見た目にずれが出る可能性がある → 許容
