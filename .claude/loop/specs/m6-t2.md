# M6-T2: IME（composition）ガード

## 目的
日本語入力の変換中にページ自動遷移が発動しないよう、EditorPage に composition 状態フラグを持ち、変換中は `splitAtLine30` 判定をスキップする。変換確定時に判定を再走させる。

## 変更対象
- ファイル:
  - `/home/cocotte/dev/diary/src/features/editor/EditorPage.tsx`
- 追加:
  - `isComposingRef = useRef(false)`
  - `onCompositionStart` / `onCompositionEnd` ハンドラを textarea に配線
  - `onChange` 内のページ遷移判定に `if (isComposingRef.current) return;` ガード
  - compositionEnd で `handleChange` 相当のロジックを再実行（最新 text で再判定）

## 依存する既存コード
- T6.1 `splitAtLine30`
- T4.3 `EditorPage` の onChange（autosave 配線）

## 実装概要（3-5行）
1. `isComposingRef = useRef(false)` を追加。
2. `onCompositionStart = () => { isComposingRef.current = true; }`。
3. `onCompositionEnd = (e) => { isComposingRef.current = false; checkOverflowAndNavigate(e.currentTarget.value); }`。
4. `onChange` で autosave は常に呼ぶが、`splitAtLine30` → 自動遷移判定は `!isComposingRef.current` の時のみ実行。
5. T5.5 の onKeyDown（PageUp/Down）も composition 中は無視する（`if (isComposingRef.current) return`）。

## 受入条件チェックリスト
- [ ] 🔵 compositionStart → onChange で 30 行を越えても遷移しない
- [ ] 🔵 compositionEnd 後に 30 行超過なら遷移する
- [ ] 🔵 confirm 無しで IME が解除された場合も遷移する（onCompositionEnd が発火）
- [ ] 🟡 textarea 単独操作（composition なし）では従来通り即判定 — 推測根拠: plan.md M6 T6.2

## Key Links（配線検証）
- `isComposingRef` は EditorPage の textarea イベント経路で直接使用。
- T6.3 の自動次ページ遷移から参照される本番配線。

## テスト方針
- ユニット (Vitest + Testing Library):
  - textarea に `fireEvent.compositionStart` → 30行超のテキスト `fireEvent.change` → navigate されない
  - 続けて `fireEvent.compositionEnd` → navigate 発火
  - composition 発火なしで 30行超の change → 即 navigate
- 目視: 日本語 IME で句読点まで長い文章を打ち、30 行境界付近で変換中に勝手にページが飛ばないか確認。

## 注意事項
- Android の GBoard など、compositionEnd が遅延・発火しないケースがある。フォールバックとして、一定時間 (例: 800ms) composition が続いたら手動で解除扱いにする、などは M6 範囲外で保留。
- `e.currentTarget` は React 19 でも event pooling が無いため安全に参照可能。
