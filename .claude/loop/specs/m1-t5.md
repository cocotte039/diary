# T1.5: 前回の続きから開く機能

## 目的
アプリを閉じて再度開いたとき、前回のカーソル位置とスクロール位置が復元され、「ノートを開いた」感覚を実現する。

## 変更対象
- `diary/src/hooks/useCursorRestore.ts` — カーソル位置保存・復元フック
- `diary/src/features/write/WritePage.tsx` — 統合

## 詳細仕様

### useCursorRestore.ts
- **保存**: textarea の `selectionStart` を localStorage に保存
  - 保存タイミング: input イベントの debounce（1秒）
  - キー: `note-cursor-position`
  - ※ `beforeunload` はモバイル Safari で不確実なため使用しない
- **復元**: アプリ起動時
  1. テキストが textarea にセットされた後
  2. localStorage から位置を読み取り
  3. `textarea.setSelectionRange(pos, pos)`
  4. 該当位置までスクロール: `textarea.scrollTop` を計算して設定

### スクロール位置の計算
```typescript
const lineNumber = text.substring(0, cursorPos).split('\n').length - 1;
const scrollTop = lineNumber * LINE_HEIGHT_EM * FONT_SIZE_PX;
textarea.scrollTop = scrollTop;
```

## 受入条件（Must-haves）

### Truths
- 🔵 テキスト入力中にカーソル位置が1秒ごとにlocalStorageに保存される
- 🔵 アプリ再起動時にカーソルが前回の位置に復元される
- 🔵 カーソル位置に応じてスクロール位置も復元される

### Artifacts
- 🔵 `src/hooks/useCursorRestore.ts` が存在する

### Key Links
- 🔵 `WritePage.tsx` が `useCursorRestore` フックを使用している

## テスト方針
- テキスト入力 → アプリ終了 → 再度開く → カーソル位置とスクロール位置が復元されることを確認
- 複数ページ分のテキストで、ページ途中のカーソル位置が復元されることを確認

## 依存関係

## 注意事項
- モバイル Safari の `beforeunload` 非対応に注意 → input イベントベースで保存
- テキスト長が変わった場合（別の場所で編集された場合）、保存位置がテキスト長を超えていたら末尾にフォールバック

## 完了サマリー

- 実変更:
  - `src/hooks/useCursorRestore.ts` (selectionStart を 1秒 debounce で localStorage 保存。復元は `restoreReady` が true かつ初回のみ)
  - `src/hooks/useDebouncedCallback.ts` (汎用 debounce フック)
  - `WritePage.tsx` で onChange/onSelect/onKeyUp/onClick から onSelectionChange を呼ぶ
- 判断:
  - beforeunload は使わず、input ベース保存（spec 指示通り）。
  - 復元時に text.length を超える保存位置は末尾にフォールバック（spec 指示通り）。
- 注意:
  - 初回ロード直後は text が空になりうるので、`restoreReady` フラグで「useWrite が ready」を待ってから復元する。
