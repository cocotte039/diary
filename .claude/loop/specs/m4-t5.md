# M4-T5: BookshelfPage のリンク先変更

## 目的
冊タップの遷移先を `/read/:id/1` から `/book/:id/:lastOpenedPage`（未設定時は最終更新ページ）に変更し、「最後に開いたページを記憶」要件を満たす。

## 変更対象
- ファイル:
  - `/home/cocotte/dev/diary/src/features/bookshelf/VolumeCard.tsx`
  - `/home/cocotte/dev/diary/src/features/bookshelf/BookshelfPage.tsx`
- 変更内容:
  - `VolumeCard` の props に `initialPage: number` を追加（親から計算済み値を受ける）
  - `to={\`/book/${volume.id}/${initialPage}\`}`
  - `BookshelfPage` 側で volumes ロード時に `lastOpenedPage` or `getLatestUpdatedPageNumber` を解決し、volume ごとの initialPage を算出

## 依存する既存コード
- `/home/cocotte/dev/diary/src/features/bookshelf/VolumeCard.tsx` 現行 `to={\`/read/${volume.id}/1\`}`
- `/home/cocotte/dev/diary/src/features/bookshelf/BookshelfPage.tsx` の `useEffect` での volumes/pages プリロード
- `/home/cocotte/dev/diary/src/lib/db.ts` `getLatestUpdatedPageNumber`（T4.2）
- `Volume.lastOpenedPage`（T4.2）

## 実装概要（3-5行）
1. `BookshelfPage` の `useEffect` 内で、volumes を取得後、各 volume に対し `initialPage = v.lastOpenedPage ?? await getLatestUpdatedPageNumber(v.id)` を並列計算（`Promise.all`）し、`Map<volumeId, number>` に格納。
2. state `initialPages` として保持。
3. `VolumeCard` に `initialPage={initialPages.get(v.id) ?? 1}` を渡す。
4. `VolumeCard` 内で `to={\`/book/${volume.id}/${initialPage}\`}` に書き換え。

## 受入条件チェックリスト
- [ ] 🔵 `lastOpenedPage` が設定されている冊のカードをタップすると `/book/:id/:lastOpenedPage` へ遷移する
- [ ] 🟡 `lastOpenedPage` 未設定の場合、最終更新ページへ遷移する — 推測根拠: plan.md C4 フォールバック
- [ ] 🔵 ページが 1 枚もない新規冊では 1 ページ目へ遷移する
- [ ] 🔵 遷移 URL が `/read/...` でなく `/book/...` に変わる

## Key Links（配線検証）
- `VolumeCard` は `BookshelfPage` の `.grid` 内で `volumes.map` から直接レンダリングされる本番経路。
- `getLatestUpdatedPageNumber` は `BookshelfPage.useEffect` から呼ばれ、ページロード時に実行される（テストだけでなく本番でも配線）。

## テスト方針
- ユニット (Vitest + Testing Library):
  - fake-indexeddb で `lastOpenedPage: 4` の volume を仕込み → `BookshelfPage` をレンダ → カードリンクが `/book/{id}/4`
  - `lastOpenedPage` 無し、ページ 1,2,3 の 2 の updatedAt が最新 → カードリンクが `/book/{id}/2`
  - 冊 0 件は別タスク（T4.6）で自動作成されるため、ここでは 1 件以上の前提
- 目視: 本棚でカードを押して遷移先 URL を確認。

## 注意事項
- initialPage 計算を同期的にしないと、カード初回描画時にリンクが `/book/:id/undefined` にならないよう、ロード完了まで `volumes` を描画しないか、初期値 1 のフォールバックを用意する。
- `Promise.all` の失敗で 1 冊分のエラーが全体ロードを止めないよう、`.catch(() => 1)` で個別 fallback する。
