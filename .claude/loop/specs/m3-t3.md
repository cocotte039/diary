# T3.3: アイコン・スプラッシュスクリーン

## 目的
ホーム画面に追加したときのアイコンとスプラッシュスクリーンを設定する。

## 変更対象
- `diary/public/` — アイコン画像追加
- `diary/public/manifest.json` — アイコン設定更新
- `diary/index.html` — apple-touch-icon meta追加

## 詳細仕様

### アイコンデザイン
- ノートのモチーフ（シンプルな横線 or ノートの角を折ったアイコン）
- ダークモードの配色に合わせた暗めの背景
- SVGで作成し、各サイズにエクスポート

### 必要なサイズ
- `icon-192x192.png` — PWA標準
- `icon-512x512.png` — PWA標準（スプラッシュ用）
- `apple-touch-icon.png`（180x180）— iOS用
- `favicon.ico`（32x32）— ブラウザタブ用

### index.html 追加
```html
<link rel="apple-touch-icon" href="/apple-touch-icon.png">
<meta name="apple-mobile-web-app-capable" content="yes">
<meta name="apple-mobile-web-app-status-bar-style" content="black-translucent">
```

## 受入条件（Must-haves）

### Truths
- 🔵 iOS/Androidでホーム画面追加時にアイコンが表示される
- 🔵 PWA起動時にスプラッシュスクリーンが表示される
- 🔵 ブラウザタブにfaviconが表示される

### Artifacts
- 🔵 `public/icon-192x192.png`, `public/icon-512x512.png`, `public/apple-touch-icon.png`, `public/favicon.ico` が存在する

## テスト方針
- iOS/Android実機でホーム画面追加時のアイコン確認
- PWA起動時のスプラッシュ表示確認

## 依存関係
- T3.2（manifest.json）

## 注意事項
- アイコン画像は手動作成 or SVGからの変換が必要
- iOS Safariのスプラッシュスクリーンは `apple-touch-startup-image` が必要な場合がある

## 完了サマリー

- 実変更:
  - `public/icon.svg` (ノート+罫線のシンプルなアイコン)
  - `public/ICONS_README.txt` (PNG 生成手順のメモ)
  - `public/manifest.json` の icons は 192/512/512-maskable を指し、未生成状態の PNG を参照
  - `index.html` に apple-touch-icon / apple-mobile-web-app-capable / status-bar-style black-translucent
- 判断:
  - PNG/ICO は本PCで生成不可（curl/ImageMagick 制約）。HANDOFF.md に ImageMagick でのワンライナーを記載し、別PCで生成を委譲。
- 注意:
  - PNG 未生成の状態で `npm run build` してもビルドは通る（VitePWA の includeAssets は存在しないファイルを警告扱い）。実運用前に生成必須。
