# M4-T4: ページ単位 autosave（useEditorAutoSave）

## 目的
`EditorPage` で編集したページ単体を 2 秒 debounce で `savePage` に保存し、`flush()` API で即時保存も可能にする。保存後は GitHub 同期を fire-and-forget。

## 実装概要（3-5行）
1. `lastSavedRef` に直近保存値、`pendingRef` に現在の保留値を保持。
2. `doSave = useDebouncedCallback(async (id, n, v) => { if (lastSavedRef.current === v) return; await savePage(id, n, v); lastSavedRef.current = v; void syncPendingPagesBackground(); }, AUTOSAVE_DEBOUNCE_MS)`。
3. `useEffect([volumeId, pageNumber, text])` で `pendingRef.current = { id, n, text }` を更新し、`doSave(id, n, text)` を呼ぶ。
4. `flush`: pendingRef の値で即時 `savePage` を同期的に await 実行（`lastSavedRef` と一致なら no-op）。debounce 中タイマーは cancel する（`useDebouncedCallback` に cancel 機能が無ければ `lastSavedRef` 更新で副作用を抑える）。
5. 戻り値 `{ flush }` を返す。

## 受入条件チェックリスト
- [ ] 🔵 text 変更から 2 秒後に `savePage` が 1 回呼ばれる
- [ ] 🔵 連続入力では最後の 1 回のみ保存（debounce）
- [ ] 🔵 `flush()` を await すると直近の text が即時保存されて resolve する
- [ ] 🟡 保存後に `syncPendingPagesBackground` が呼ばれる — 推測根拠: 既存 `useAutoSave` と同じ挙動
- [ ] 🔵 同じ text の 2 連続保存は no-op（`lastSavedRef` 比較）

## テスト方針
- ユニット (Vitest + fake timers):
  - `renderHook` で hook を起動、text を変える → `vi.advanceTimersByTime(2000)` → `savePage` が 1 回呼ばれる
  - 連続 text 変更で最後の 1 回のみ保存
  - `flush()` 呼び出しで直ちに `savePage` が発火し、その後の timer fire では重複保存されない
  - 同じ値を 2 回投入しても 2 回目は no-op
- TDD 推奨: Red (テスト書く) → Green (実装) → Refactor。

## 注意事項
- `useDebouncedCallback` の現行実装に cancel がない場合は、`lastSavedRef` 比較で重複保存を防ぐ形に倒す。必要なら同フックに `cancel()` を追加する（本タスクで許容）。
- `flush` 中に text が変更される race は、pendingRef を `flush` 呼び出し直前に読む設計で簡易対応。
- 既存 `useAutoSave` は WritePage 用に残す（M7 で WritePage ごと削除）。本フックは分離することで互換事故を避ける。
