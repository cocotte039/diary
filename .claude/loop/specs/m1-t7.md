# M1-T7: 配線検証 + Verify チェックリスト

## 目的

新仕様の配線が完全に切り替わっているかを grep で検証し、実機 Verify 観点のチェックを記録する。

## 変更対象

なし（検証のみ）。

## 実装概要

### grep 配線検証（自動）

```bash
# 削除されているべき参照
grep -r checkOverflowAndNavigate src/   # 0 件
grep -r pendingCursorPosRef src/        # 0 件
grep -r handleBeforeInput src/features/editor/  # 0 件
grep -r splitAtCharLimit src/           # 0 件
grep onBeforeInput src/features/editor/EditorPage.tsx  # 0 件

# 残っているべき参照
grep -n isComposingRef src/features/editor/EditorPage.tsx  # 複数件（スワイプ・キー遷移・composition）
grep -n CHARS_PER_PAGE src/features/editor/EditorPage.tsx  # progressPct 計算のみ
grep "25 \* var" src/styles/notebook.css  # 2 件（background-image + background-size）
```

### 実機 Verify チェックリスト（手動、ユーザー確認）

plan.md D10 の全項目を記録する。

#### 機能面
- [ ] 1200 字超の入力でページ遷移が起きない
- [ ] 最終ページで 1201 字を書き込める
- [ ] 日付スタンプ挿入で 1200 字超えても現ページに留まる
- [ ] 進捗バーが 1200 字到達で 100%、それ以降変化しない
- [ ] 手動ページ送り（スワイプ / 矢印ボタン / PageUp/PageDown）が従来通り動作
- [ ] 既存ページ（1200 字以下）が従来通り表示・編集できる

#### 視覚面
- [ ] 25 行ごと罫線が視認できる（濃すぎず薄すぎず）
- [ ] 強調罫線が通常罫線の Y 座標と重なる（1 px ズレなし）
- [ ] スクロール時に罫線がテキストと同期する

#### パフォーマンス面
- [ ] 5000 字入力時の入力遅延なし
- [ ] 10000 字入力時の入力遅延なし

## 受入条件チェックリスト

- [ ] 🔵 grep 配線検証が全て期待通り
- [ ] 🟡 実機 Verify チェックリストの機能面 6 項目が OK（iOS Safari / Android Chrome 想定。本 /run 自動実行中は目視不可なので、AGENTS.md に「ユーザー実機確認待ち」として記録）
- [ ] 🔵 軽微な不備（色濃度等）があれば自己修正（最大 3 回）
- [ ] 🔵 重大な問題があれば fix_queue に追加
