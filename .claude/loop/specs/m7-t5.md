# M7-T5: WritePage / ReaderPage 削除

## 目的
EditorPage に機能統合済みの WritePage / ReaderPage とその関連コード（useWrite 等）を削除し、App.tsx のルートも整理する。

## 変更対象
- ファイル（削除）:
  - `/home/cocotte/dev/diary/src/features/write/` ディレクトリ全体
  - `/home/cocotte/dev/diary/src/features/reader/` ディレクトリ全体
- ファイル（修正）:
  - `/home/cocotte/dev/diary/src/App.tsx`: `/write` と `/read/:id/:page` のルートを削除。ただし `/read/:volumeId/:pageNumber` は `/book/...` への redirect として残す（T4.1 で追加した `ReadRedirect` は保持）。
  - `src/` 配下で `useWrite` / `WritePage` / `ReaderPage` を import しているファイルを掃除

## 依存する既存コード
- T4.1 で追加した `/read/:...` → `/book/:...` redirect
- T7.4 までで EditorPage が WritePage/ReaderPage の全機能を吸収済み

## 実装概要（3-5行）
1. `rg "WritePage|ReaderPage|useWrite"` で依存箇所をすべて列挙。
2. `App.tsx` から `/write` ルートを削除、imports から WritePage を除去。ReaderPage import も除去（redirect は別コンポーネントで継続）。
3. `rm -rf src/features/write src/features/reader`（Git 経由）。
4. 残存 import の TypeScript エラーをすべて解消（関連テストファイルがあれば削除または移行）。
5. `npm run build` & `npm run test` が通ることを確認。

## 受入条件チェックリスト
- [ ] 🔵 `src/features/write/` が存在しない
- [ ] 🔵 `src/features/reader/` が存在しない
- [ ] 🔵 `npm run build` がエラー無しで通る
- [ ] 🔵 `npm run test` が全パス
- [ ] 🔵 `/#/read/:id/:page` 形式の URL が `/#/book/:id/:page` にリダイレクトされる（T4.1 の redirect 継続）
- [ ] 🟡 旧 `/write` URL は 404 ではなく `/`（本棚）にフォールバックする — 推測根拠: `path="*"` の Navigate to="/"

## Key Links（配線検証）
- `App.tsx` は main.tsx から呼ばれる本番経路。
- redirect 用コンポーネント（`ReadRedirect` 相当）は T4.1 で既に配線済み。本タスクでも残す。

## テスト方針
- ユニット (Vitest):
  - `/book/:id/:page` への redirect テストが T4.1 から引き続きパス
  - `/write` にアクセス → `/` にリダイレクト（`path="*"` フォールバック）
  - 削除後の tree で `features/write` `features/reader` がない（`fs.existsSync` check は CI で実施可）
- 目視: 本番ビルド後に旧 URL ブックマークが機能する。

## 注意事項
- `useWrite` のテストファイルが残ると参照エラーになるので同時削除。
- `WritePage.module.css` を他コンポーネントが import していないか確認（基本は WritePage のみの想定）。
- main.tsx / index.html / PWA manifest から `/write` 経路の参照がないか最終確認。
