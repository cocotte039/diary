# M6-T1: `splitAtLine30` 純関数追加

## 目的
30 行境界で「このページに残す部分」と「次ページへ持ち越す部分」を分離する純関数を `pagination.ts` に追加する。自動ページ送り（T6.3）とロック判定（T6.4）の基礎。

## 変更対象
- ファイル:
  - `/home/cocotte/dev/diary/src/lib/pagination.ts`
  - `/home/cocotte/dev/diary/src/lib/pagination.test.ts`（既存に追記）
- 関数（新規）:
  - `splitAtLine30(text: string): { keep: string; overflow: string }`
  - 仕様: `keep` = 最初の 30 行を `\n` 結合、`overflow` = 31 行目以降を `\n` 結合。行数 ≤ 30 なら `overflow === ''`。
- 定数:
  - `LINES_PER_PAGE` を利用（30）

## 依存する既存コード
- `/home/cocotte/dev/diary/src/lib/constants.ts` `LINES_PER_PAGE`
- `/home/cocotte/dev/diary/src/lib/pagination.ts` 既存 `splitIntoPages` のロジック（参考）

## 実装概要（3-5行）
1. `const lines = text.split('\n')`。
2. `if (lines.length <= LINES_PER_PAGE) return { keep: text, overflow: '' };`。
3. `const keep = lines.slice(0, LINES_PER_PAGE).join('\n');`。
4. `const overflow = lines.slice(LINES_PER_PAGE).join('\n');`。
5. `return { keep, overflow };`。

## 受入条件チェックリスト
- [ ] 🔵 空文字入力で `{ keep: '', overflow: '' }`
- [ ] 🔵 29 行で `overflow === ''`、keep が原文と同じ
- [ ] 🔵 30 行でも `overflow === ''`、keep が原文と同じ
- [ ] 🔵 31 行で keep は 30 行、overflow は 1 行分
- [ ] 🔵 45 行で keep 30 行、overflow 15 行
- [ ] 🔵 round-trip: overflow 非空時、`keep + '\n' + overflow === 元テキスト`

## 注意事項
- `text === ''` のとき `split('\n')` は `['']`（length=1）なので早期リターンで OK。
- CR/LF は入力段階で `\n` 統一されている前提（既存 textarea 動作と同じ）。必要なら関数内で `.replace(/\r\n?/g, '\n')` を事前に入れても可。
