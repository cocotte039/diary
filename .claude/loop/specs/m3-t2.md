# T3.2: PWAオフライン対応

## 目的
Workbox設定を拡充し、オフラインでもアプリが完全に動作するようにする。Google Fontsのキャッシュも含む。

## 変更対象
- `diary/vite.config.ts` — PWAプラグイン設定拡充
- `diary/public/manifest.json` — PWAマニフェスト設定

## 詳細仕様

### vite.config.ts — Workbox設定
```typescript
VitePWA({
  registerType: 'autoUpdate',
  workbox: {
    // プリキャッシュ: HTML, JS, CSS
    globPatterns: ['**/*.{js,css,html,ico,png,svg}'],
    // ランタイムキャッシュ: Google Fonts
    runtimeCaching: [
      {
        urlPattern: /^https:\/\/fonts\.googleapis\.com\/.*/i,
        handler: 'CacheFirst',
        options: {
          cacheName: 'google-fonts-cache',
          expiration: { maxEntries: 10, maxAgeSeconds: 60 * 60 * 24 * 365 },
          cacheableResponse: { statuses: [0, 200] },
        },
      },
      {
        urlPattern: /^https:\/\/fonts\.gstatic\.com\/.*/i,
        handler: 'CacheFirst',
        options: {
          cacheName: 'gstatic-fonts-cache',
          expiration: { maxEntries: 10, maxAgeSeconds: 60 * 60 * 24 * 365 },
          cacheableResponse: { statuses: [0, 200] },
        },
      },
    ],
  },
  manifest: false, // 手動でmanifest.jsonを管理
})
```

## 受入条件（Must-haves）

### Truths
- 🔵 オフライン時もアプリが起動・表示される
- 🔵 オフライン時もフォントが表示される（キャッシュ済み）
- 🔵 オフライン時も日記の書き込み・保存（IndexedDB）ができる
- 🔵 Lighthouse PWAスコアが基本要件を満たす

### Artifacts
- 🔵 `public/manifest.json` が存在する
- 🔵 Service Worker が登録されている

## テスト方針
- DevTools Network タブで Offline モードにし、アプリ動作確認
- フォントがオフラインで正しく表示されることを確認
- Lighthouse PWA 監査実行

## 注意事項
- manifest.json の `start_url` はGitHub Pagesのサブディレクトリに注意（`/repo-name/` になる場合がある）
- Service Workerの更新時、キャッシュの整合性に注意

## 完了サマリー

- 実変更:
  - `vite.config.ts` (VitePWA runtimeCaching: Google Fonts の CacheFirst × 2、globPatterns、manifest: false)
  - `public/manifest.json` (name/short_name "ノート"、lang ja、display standalone、theme/background #1c1c20、icons 3エントリ)
- 判断:
  - manifest は public 配下で手動管理（spec 指示: `manifest: false`）。VitePWA が include assets 扱いする。
  - start_url/scope は `./` とし、GitHub Pages のサブディレクトリでも追随する（vite.config.ts の base と連携）。
- 注意:
  - 実 Workbox の検証はビルド後 SW で行う。別PCで `npm run build && npm run preview` 後に DevTools Application > Service Workers を確認。
