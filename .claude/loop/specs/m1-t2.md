# T1.2: ノート風の書く画面

## 目的
フルスクリーンの textarea を手書き風フォント＋罫線背景でスタイリングし、「ノートに書いている」見た目を実現する。

## 変更対象
- `diary/src/features/write/WritePage.tsx` — 書く画面コンポーネント
- `diary/src/features/write/WritePage.module.css` — 書く画面スタイル
- `diary/src/styles/notebook.css` — ノート風共通スタイル（罫線・フォント）

## 詳細仕様

### WritePage.tsx
- フルスクリーン textarea（100vw x 100vh - キーボード分）
- textarea属性:
  - `autoComplete="off"`
  - `autoCorrect="off"`
  - `spellCheck={false}`
  - `autoCapitalize="off"`
- 余計なUI要素なし（この段階ではtextareaのみ）

### notebook.css — 罫線の実装
```css
.notebook-textarea {
  font-family: var(--font-family);
  font-size: var(--font-size);
  line-height: var(--line-height);
  letter-spacing: var(--letter-spacing);
  color: var(--color-text);
  background-color: var(--color-bg);
  /* 罫線: line-height に合わせた repeating-linear-gradient */
  background-image: repeating-linear-gradient(
    transparent,
    transparent calc(var(--line-height) * var(--font-size) - 1px),
    var(--color-rule) calc(var(--line-height) * var(--font-size) - 1px),
    var(--color-rule) calc(var(--line-height) * var(--font-size))
  );
  background-size: 100% calc(var(--line-height) * var(--font-size));
  /* テキストと罫線の位置合わせ */
  padding-top: /* フォントのベースラインに合わせて調整 */;
  border: none;
  outline: none;
  resize: none;
  width: 100%;
  caret-color: var(--color-text);
}
```

### フォントロード検証（Critical リスク対応）
- `font-display: swap` によるFOUT発生時、line-heightが変わると罫線とテキストがずれる
- フォールバックフォント（sans-serif）との line-height 差を測定
- 必要に応じて `font-size-adjust` or `@font-face` の `ascent-override`, `descent-override` で調整

### ダークモード配色（暫定値）
- 背景: `#1c1c20`（暖かみのある黒）
- テキスト: `#e8e0d4`（暖かみのあるオフホワイト）
- 罫線: `rgba(255, 255, 255, 0.08)`（極めて薄い白）
- WCAGコントラスト比 4.5:1 以上を確保

## 注意事項
- CSS background-size と line-height の同期が崩れると全画面のレイアウトが壊れる → CSS 変数で一元管理
- フォントの line-height 調整は最も重要な作業。ここで妥協すると後工程すべてに影響する
