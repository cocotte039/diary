# M1-T2: 戻るボタンガード実装 + 追加テスト（TDD）

## 目的

EditorPage 表示中に Android 端末の戻るボタン（= `popstate` イベント）が発火したとき、前の履歴ではなく本棚 (`/`) に直接戻る挙動を実現する。ページめくり後でも常に本棚へ一本化する。

- `src/features/editor/EditorPage.test.tsx` (新規 describe ブロック追加、約 60 行)

## 依存する既存コード

- `const navigate = useNavigate()` (L64)
- `const { flush } = useEditorAutoSave(...)` (L156)
- `const historyGuardInstalledRef = useRef(false)` を新規追加

## 新規関数（useEffect 内ハンドラ）

`useEffect` 内に定義されるため個別関数ではないが、以下の内容:

```tsx
const onPopState = () => {
  void flush().catch(() => {});
  navigate('/', { replace: true });
};
```

## 実装概要

`EditorPage.tsx` 冒頭（既存 useEffect 群の後、`return (...)` の前）に新 useEffect を追加:

```tsx
// 戻るボタンガードの二重 pushState を防ぐ ref (StrictMode 対策)
const historyGuardInstalledRef = useRef(false);

useEffect(() => {
  // Android 端末の戻るボタン（popstate）で本棚 (`/`) に戻すためのガード。
  // マウント時にダミー履歴を 1 件積み、popstate で flush → navigate('/', replace) する。
  // StrictMode 二重マウントに備え pushState は ref で 1 回に制限する。
  if (!historyGuardInstalledRef.current) {
    window.history.pushState({ editorGuard: true }, '');
    historyGuardInstalledRef.current = true;
  }
  const onPopState = () => {
    void flush().catch(() => {});
    navigate('/', { replace: true });
  };
  window.addEventListener('popstate', onPopState);
  return () => {
    window.removeEventListener('popstate', onPopState);
  };
}, [flush, navigate]);
```

## テスト方針（Red→Green→Refactor: TDD）

**必ずテストを先に書く。** 新規 describe を追加し、4 件のケースを書いて RED を確認してから実装に進む。

```tsx
describe('EditorPage back button guard (popstate → 本棚)', () => {
  it('戻るボタン(popstate) で / に navigate する', async () => {
    const v = await ensureActiveVolume();
    let pathname = '';
    renderWithLocationProbe(`/book/${v.id}/3`, (p) => { pathname = p; });
    await screen.findByLabelText('日記本文');
    act(() => {
      window.dispatchEvent(new PopStateEvent('popstate'));
    });
    await waitFor(() => expect(pathname).toBe('/'));
  });

  it('ページめくり後の戻るボタンでも / に戻る', async () => {
    const v = await ensureActiveVolume();
    let pathname = '';
    renderWithLocationProbe(`/book/${v.id}/1`, (p) => { pathname = p; });
    await screen.findByLabelText('日記本文');
    fireEvent.click(screen.getByRole('button', { name: '次のページ' }));
    await waitFor(() => expect(pathname).toBe(`/book/${v.id}/2`));
    act(() => {
      window.dispatchEvent(new PopStateEvent('popstate'));
    });
    await waitFor(() => expect(pathname).toBe('/'));
  });

  it('戻るボタン発火前の編集内容が flush で保存される', async () => {
    const v = await ensureActiveVolume();
    renderAt(`/book/${v.id}/1`);
    const textarea = (await screen.findByLabelText('日記本文')) as HTMLTextAreaElement;
    fireEvent.change(textarea, { target: { value: 'draft-before-back' } });
    act(() => {
      window.dispatchEvent(new PopStateEvent('popstate'));
    });
    await waitFor(async () => {
      const saved = await getPage(v.id, 1);
      expect(saved?.content).toBe('draft-before-back');
    });
  });

  it('アンマウント後の popstate では navigate が呼ばれない', async () => {
    const v = await ensureActiveVolume();
    let pathname = '';
    const { unmount } = renderWithLocationProbe(`/book/${v.id}/1`, (p) => { pathname = p; });
    await screen.findByLabelText('日記本文');
    unmount();
    pathname = '(cleared)';
    act(() => {
      window.dispatchEvent(new PopStateEvent('popstate'));
    });
    expect(pathname).toBe('(cleared)');
  });
});
```

必要なヘルパー:
- `renderWithLocationProbe(path, onLocationChange)`: MemoryRouter でレンダリングし、LocationProbe が `useLocation().pathname` を `onLocationChange` に渡す
- 既存の `renderAt` / `ensureActiveVolume` / `getPage` は利用可能

## 注意事項

- **HashRouter と MemoryRouter の違い**: 本番は HashRouter だがテストは MemoryRouter。MemoryRouter は window.popstate を直接は聞かないが、我々のリスナが `navigate('/')` を呼ぶことで MemoryRouter 内部 history に反映される → LocationProbe で観測可能
- **StrictMode ガード**: `historyGuardInstalledRef` は useEffect 外のフック領域で `useRef(false)` として宣言。useEffect の二重実行で pushState が二重にならないように
- **cleanup でのダミー履歴撤去は不要**: `history.back()` を cleanup で呼ぶと新たな popstate が発火し、意図しない navigate を誘発する恐れがある。1 件ゴミが残るが実害なし
- **静けさ原則**: popstate 時にトースト・確認ダイアログを出さない（即座に navigate）
- **deps 配列**: `[flush, navigate]`。どちらも安定参照
- **import**: `useRef` は既に import 済み、追加不要
