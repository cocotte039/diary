# M5-T2: 180ms フェードトランジション

## 目的
EditorPage のページ切替時に 180ms の opacity フェードを入れ、紙のめくり感を損なわず遷移を視覚化する。要件 180ms と既存 200ms (`--transition-soft`) を CSS 変数分離で共存。

## 変更対象
- ファイル:
  - `/home/cocotte/dev/diary/src/styles/global.css`（`--transition-page` 追加）
  - `/home/cocotte/dev/diary/src/features/editor/EditorPage.tsx`（fading state）
  - `/home/cocotte/dev/diary/src/features/editor/EditorPage.module.css`（fading class）
- CSS 変数:
  - `--transition-page: opacity 180ms ease;`（新規、編集ページ専用）
  - `--transition-soft: opacity 200ms ease;`（既存維持、本棚等で利用）
- 関数:
  - 既存 `goPage` を「fading=true → 180ms 後 navigate」に拡張、または ReaderPage 同等の `setTimeout` パターン

## 依存する既存コード
- `/home/cocotte/dev/diary/src/features/reader/ReaderPage.tsx` の `fading` 機構（移植元）
- `/home/cocotte/dev/diary/src/styles/global.css` の `:root` 変数群

## 実装概要（3-5行）
1. `global.css` の `:root` に `--transition-page: opacity 180ms ease;` を追加。
2. `EditorPage.module.css` に `.surface { transition: var(--transition-page); }` と `.fading { opacity: 0; }`。
3. `EditorPage.tsx` に `const [fading, setFading] = useState(false)` を追加。
4. `goPage(delta)`: `setFading(true)` → `await flush()` → `await updateVolumeLastOpenedPage(...)` → `setTimeout(() => { navigate(...); setFading(false); }, 180)`。
5. textarea ラッパに `${fading ? styles.fading : ''}` を付与。

## 受入条件チェックリスト
- [ ] 🔵 ページ遷移時に opacity 1→0→1 のフェードが 180ms で起こる
- [ ] 🟡 既存の `--transition-soft` (200ms) は本棚など他ページで影響を受けない — 推測根拠: 別変数化方針
- [ ] 🔵 フェード中も flush は完了している（データロスが起きない）
- [ ] 🟡 textarea のフォーカスがフェード後に復帰する — 推測根拠: ReaderPage と同等の体験

## Key Links（配線検証）
- `EditorPage` の `goPage` は T5.1 のボタン、T5.3 のスワイプ、T5.5 のキー、T6.3 の自動遷移から呼ばれる。本タスクの fading 機構はそれらすべてで効く。

## テスト方針
- ユニット (Vitest + fake timers):
  - 「次へ」クリック → 即座に `fading` クラスが付き、180ms 進めると navigate が呼ばれ、`fading` クラスが外れる
  - 同期的タイミング検証: `flush` 完了 → setTimeout(180) → navigate
- 目視: 実機でページ送り時のフェード時間が「ふわっと」自然な短さ（180ms）であることを確認。

## 注意事項
- React 19 では `useEffect` のクリーンアップで `setFading(false)` を呼ぶ際の race に注意。`cancelled` フラグまたは ref で対応。
- フェード中の連続クリック（ダブルクリック）は無視する（`if (fading) return` ガード）。
- 既存 ReaderPage は M7 で削除されるが、それまで `--transition-soft` を参照し続ける。本変更で互換性は保たれる。
