# T3.4: フォント・カラー・罫線の微調整

## 目的
実機での検証結果を元に、配色・罫線・フォント・全体の一貫性を最終調整する。

## 変更対象
- `diary/src/lib/constants.ts` — 数値の微調整
- `diary/src/styles/global.css` — CSS変数の微調整
- `diary/src/styles/notebook.css` — 罫線スタイルの微調整
- 各 `.module.css` — 必要に応じて微調整

## 詳細仕様

### 調整項目
1. **配色の暖かみ調整**
   - WCAG AA コントラスト比 4.5:1 以上を確保しつつ温かみを出す
   - 暗い部屋での目への負担を最小化（輝度を抑える）

2. **罫線の太さ・透明度**
   - 実機で見て「邪魔にならないが存在を感じる」程度に調整

3. **ページ区切り線のスタイル**
   - 罫線との差異が適切か確認
   - 冊終わりの色味変化が自然か確認

4. **日付入力補助ボタンのデザイン**
   - 目立たなさと操作性のバランス

5. **全画面の一貫性チェック**
   - 書く画面、本棚、読み返し、設定の統一感
   - フォントサイズ、余白、カラーの統一

6. **トランジション・アニメーション**
   - 画面遷移: 200ms ease
   - ページめくり: opacity 200ms ease
   - 過度なアニメーションがないか確認

## 受入条件（Must-haves）

### Truths
- 🔵 全画面でWCAG AAコントラスト比 4.5:1 以上
- 🔵 暗い部屋で目への負担が少ない
- 🔵 全画面で配色・フォント・余白が統一されている
- 🔵 アニメーションが控えめで「静けさ」を壊さない

## テスト方針
- スマホ実機で全画面を通して暗い部屋で確認
- WCAG コントラスト比チェッカーで確認
- 複数ユーザーへの主観的な印象確認（可能であれば）

## 依存関係
- 全タスクの後（最終調整）

## 注意事項
- Phase 1 で設定した暫定値からの変更なので、影響範囲を確認
- CSS変数で一元管理しているため、変更は比較的安全

## 完了サマリー

- 実変更:
  - `src/styles/global.css` の CSS 変数に `--color-error`, `--color-page-divider-end`, `--line-height-px` を集約
  - `src/styles/notebook.css` の `.notebook-surface` で line-height と background-size を `--line-height-px` で同期
  - `notebook-reader` クラスを追加し読み返し画面と書く画面で見た目を完全共通化
- 判断:
  - 配色は暫定値のまま（spec は「実機での検証結果を元に」微調整。別PCの実機確認後に再調整）。AGENTS.md に「暫定値→本PC環境では実機確認不能のため未調整」と記載。
  - WCAG コントラスト比: 背景 #1c1c20 とテキスト #e8e0d4 は約 12:1 で AA 余裕クリア（机上計算）。
- 注意:
  - CSS 変数一元管理なので、実機後に変更する場合は global.css のみ編集すれば全画面に反映される。
