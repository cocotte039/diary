# T3.1: GitHub APIバックアップ

## 目的

### 同期トリガー
1. ローカル保存成功後、非同期でGitHub同期を実行
2. オフライン時: `syncStatus` を `'pending'` のまま維持
3. `online` イベントでキューイングされた未同期ページをまとめてコミット
4. エラー時: リトライ（最大3回、exponential backoff: 1s, 2s, 4s）

### エラーハンドリング
- 401: トークン無効 → 設定画面にエラー表示（控えめ）
- 404: リポジトリ未作成 → ユーザーに作成を促す
- 422: SHA不一致（コンフリクト） → 最新SHAを取得してリトライ
- ネットワークエラー: `'pending'` のまま維持、次回同期時にリトライ

## 注意事項
- トークンのスコープ: `repo` のみ必要
- GitHub API レート制限: 認証済みで 5000 req/hr（個人利用なら十分）
- 大量の未同期ページがある場合、まとめてコミットするとAPI呼び出し回数が多くなる → バッチコミット（1回のAPIコールで複数ファイル更新）を検討

- 実変更:
  - `src/lib/github.ts` (Octokit による syncPage/syncPendingPages/testConnection/registerOnlineSync。SHA in-memory cache、exponential backoff、422再取得)
  - `useAutoSave` が保存後に `syncPendingPagesBackground()` を fire-and-forget

  - `main.tsx` は initStoragePersistence のみ起動、`registerOnlineSync` は WritePage で登録
- 判断:
  - トークン保存先は IndexedDB (meta ストア)。理由: localStorage は eviction に弱く、iOS Safari で消えやすい。
  - ファイルパス: `volumes/{ordinal(3桁0詰め)}-{volumeId}/page-{pageNumber(2桁)}.txt`（昇順に並ぶ）。
  - UTF-8 → base64 は TextEncoder 経由（btoa は Latin-1 のみのため）。
- 注意:
  - GitHub API レート: 認証済み 5000 req/hr。1保存ごとに PUT 1 回で個人利用では余裕。
  - 422 conflict は SHA キャッシュを破棄し次回リトライでは最新 SHA を使う。
