## 実装概要（TDD）

1. **Red**: 同時刻 `createdAt` の 3 冊（ordinal 1/2/3）を投入し、レンダー後に `[3, 2, 1]` 順になることを assert するテストを追加 → 失敗
2. **Green**: ソート関数を ordinal 降順（tie-break で createdAt 降順）に変更
3. **Refactor**: 型定義の JSDoc 追記

## 変更後コード

`BookshelfPage.tsx` L44 付近:

```ts
// ordinal 降順（最新作成を上）。
// 🟡 J7: 同 ordinal はデータ異常時の保険として createdAt で tie-break。
vs.sort((a, b) => {
  if (b.ordinal !== a.ordinal) return b.ordinal - a.ordinal;
  return b.createdAt.localeCompare(a.createdAt);
});
```

## 追加テスト

既存の DB 操作ヘルパ（`ensureActiveVolume`・`rotateVolume`）や `idb` 直操作で同時刻 createdAt の 3 冊を投入し、並び順が `[3, 2, 1]` になることを確認:

```ts
it('同時刻 createdAt でも ordinal 降順で並ぶ', async () => {
  const sameTime = new Date('2026-04-15T12:00:00.000Z').toISOString();
  // db ストア直接に 3 冊書き込み（既存テストのパターンに合わせる）
  const { getDB } = await import('../../lib/db');
  const db = await getDB();
  const tx = db.transaction('volumes', 'readwrite');
  await tx.store.put({ id: 'v1', createdAt: sameTime, status: 'completed', ordinal: 1 });
  await tx.store.put({ id: 'v2', createdAt: sameTime, status: 'completed', ordinal: 2 });
  await tx.store.put({ id: 'v3', createdAt: sameTime, status: 'active',    ordinal: 3 });
  await tx.done;
  renderPage();
  const links = await screen.findAllByRole('link', { name: /ノート \d+/ });
  const ordinals = links.map((el) => el.textContent?.match(/ノート (\d+)/)?.[1]);
  expect(ordinals.slice(0, 3)).toEqual(['3', '2', '1']);
});
```

※ 既存テストでの DB 操作パターンに合わせて書き換え可。
