# M6-T5: 本棚に「新しい冊」ボタン追加

## 目的
本棚グリッド末尾に破線境界の「＋ 新しい冊」カードを追加し、タップで confirm ダイアログ経由 `rotateVolume` を呼び新冊を作成する（plan.md 未解決事項 4: A 案採用）。

## 変更対象
- ファイル:
  - `/home/cocotte/dev/diary/src/features/bookshelf/BookshelfPage.tsx`
  - `/home/cocotte/dev/diary/src/features/bookshelf/BookshelfPage.module.css`
  - 新規: `/home/cocotte/dev/diary/src/features/bookshelf/NewVolumeCard.tsx`（or BookshelfPage 内インライン）
- 追加コンポーネント:
  - `NewVolumeCard` — 破線境界の button、`aria-label="新しい冊を作る"`、中央に `＋` 表記
- 関数:
  - `handleCreateNew()`: active 冊のページ数を計算 → `window.confirm(\`現在の冊は ${count} / 50 ページです。新しい冊を作りますか？\`)` → true なら `rotateVolume(activeId)` → 再ロード

## 依存する既存コード
- `/home/cocotte/dev/diary/src/lib/db.ts` `rotateVolume(currentActiveId)`, `getActiveVolume()`, `getPagesByVolume()`
- `/home/cocotte/dev/diary/src/features/bookshelf/BookshelfPage.module.css` `.grid`
- 既存 `volumes.length === 0` 時は NewVolumeCard を非表示（T4.6 が自動作成する）

## 実装概要（3-5行）
1. `NewVolumeCard` を `.grid` の末尾にレンダ（`volumes.length > 0` 条件）。
2. クリック時 `onCreate()` を親 `BookshelfPage` の handler にバブル。
3. `handleCreateNew`: `const active = await getActiveVolume();` → 無ければ早期 return。`const pages = await getPagesByVolume(active.id);` → `count = pages.length`。
4. `if (window.confirm(\`現在の冊は ${count} / 50 ページです。新しい冊を作りますか？\`)) { await rotateVolume(active.id); /* reload */ }`
5. reload: `getAllVolumes()` / `getAllPages()` を再実行し setState。

## テスト方針
- ユニット (Vitest + Testing Library):
  - DB に active volume 1 件、ページ 3 件 → BookshelfPage レンダ → NewVolumeCard が見える
  - `window.confirm` をモックし true → カードクリック → `rotateVolume` が呼ばれ、その後 volumes 数が 2 になる
  - confirm false → `rotateVolume` 呼ばれない
  - 冊 0 件 (ensureActiveVolume 前提) では NewVolumeCard が non-existent
  - confirm メッセージに "3 / 50" が含まれる
- 目視: 実機で破線カード見た目とダイアログ文言確認。

## 注意事項
- `window.confirm` はモバイルでブラウザ UI が出る（PWA 標準）。独自モーダル化は将来課題。
- 破線 CSS 例: `border: 1px dashed var(--color-rule); opacity: 0.5;`。active カードと視覚的に区別できるようにする。
- rotateVolume 後に新冊が active になる。旧冊 `lastOpenedPage` は維持される（rotateVolume 内で触らない）。
