# M4-T3: 最小 EditorPage 新規作成

## 目的
1ページ=1 textarea の独立 UI として `EditorPage` を新設する。URL `/book/:volumeId/:pageNumber` で特定ページを表示・編集可能にする。

## 変更対象
- ファイル（新規）:
  - `/home/cocotte/dev/diary/src/features/editor/EditorPage.tsx`
  - `/home/cocotte/dev/diary/src/features/editor/EditorPage.module.css`
- コンポーネント:
  - `EditorPage`: default export, `useParams<{ volumeId: string; pageNumber: string }>()` で受け取る
- ヘッダー要素:
  - 左: `<Link to="/" aria-label="本棚に戻る">本棚</Link>`
  - 中央: `<div>{current} / {PAGES_PER_VOLUME}</div>`
  - 右: `<Link to="/settings">設定</Link>`

## 依存する既存コード
- `/home/cocotte/dev/diary/src/lib/db.ts` `getPage(volumeId, pageNumber)`
- `/home/cocotte/dev/diary/src/lib/constants.ts` `PAGES_PER_VOLUME`
- `/home/cocotte/dev/diary/src/styles/notebook.css` の `.notebook-surface` クラス
- `/home/cocotte/dev/diary/src/features/write/WritePage.module.css` の textarea スタイル（参考、流用は最小限）

## 実装概要（3-5行）
1. `useParams` から volumeId, pageNumber（Number 化、NaN/範囲外は 1 にフォールバック）を取得。
2. `useEffect([volumeId, current])` で `getPage()` を呼び `setText(page?.content ?? '')`。ロード完了フラグ `ready` を立てる。
3. textarea (`notebook-surface` クラス併用) を表示。`onChange` で local state 更新、T4.4 の `useEditorAutoSave(volumeId, current, text)` を呼ぶ（autosave 配線）。
4. CSS は最低限: textarea 幅100% / 高さ可変 / 罫線整合は M7 T7.1-T7.2 で詰める。
5. ヘッダーは Aesthete 案の3要素配置（左=本棚 / 中央=ページ番号 / 右=設定）。

## 受入条件チェックリスト
- [ ] 🔵 `/book/:volumeId/3` にアクセスすると 3 ページ目の content が textarea に表示される
- [ ] 🔵 textarea で編集すると local state が更新される
- [ ] 🔵 ヘッダー左の「本棚」リンクで `/` へ戻れる
- [ ] 🔵 ヘッダー中央に `current / 50` が表示される
- [ ] 🟡 pageNumber が範囲外 (0, 51, NaN) のときは 1 ページ目を表示する — 推測根拠: 安全側フォールバック

## Key Links（配線検証）
- `EditorPage` は T4.1 で `App.tsx` の `<Route path="/book/:volumeId/:pageNumber">` から直接レンダリングされる本番コンポーネント。
- T4.4 の `useEditorAutoSave` を本コンポーネントから呼び出すことで autosave が配線される。テスト専用の呼び出しで終わらせない。

## テスト方針
- ユニット (Vitest + Testing Library + MemoryRouter):
  - fake-indexeddb で volume + page #3 を仕込み → `<MemoryRouter initialEntries={['/book/v1/3']}>` でレンダ → ページの content が textarea に出る
  - 存在しないページ番号にアクセスで textarea 空文字
- 目視: 実機で `/#/book/{id}/1` を開き、罫線が見える / 編集できる。

## 注意事項
- M4 時点では 30行ロックや自動次ページ遷移は実装しない（M6）。
- 罫線整合（ヘッダー重なり）の最終調整は M7 T7.2 まで持ち越し。M4 では最低限の見た目に留める。
- `notebook-surface` クラスはグローバル `src/styles/notebook.css` 由来なので `import '../styles/notebook.css'` の経路は既存どおり（main.tsx で読み込み済みであれば不要）。
