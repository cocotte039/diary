# T8-2.1: EditorPage スワイプ B 案化（textarea 上でもページめくり）

## 変更対象
- `src/features/editor/EditorPage.tsx`

## 実装概要
`isFromTextarea` ガードを撤廃。textarea 上でも左右スワイプでページ遷移できるようにする。水平優位判定を 2:1 に強化し、縦スクロールや改行入力との干渉を避ける。IME 変換中のスワイプは無視。

## 差分イメージ

```tsx
// before
const isFromTextarea = (target: EventTarget | null): boolean =>
  target instanceof HTMLTextAreaElement;

const onTouchStart = (e: ReactTouchEvent<HTMLDivElement>) => {
  if (isFromTextarea(e.target)) {
    touchStartXRef.current = null;
    touchStartYRef.current = null;
    return;
  }
  const t = e.touches[0];
  if (!t) return;
  touchStartXRef.current = t.clientX;
  touchStartYRef.current = t.clientY;
};

const onTouchEnd = (e: ReactTouchEvent<HTMLDivElement>) => {
  if (isFromTextarea(e.target)) return;
  const startX = touchStartXRef.current;
  const startY = touchStartYRef.current;
  touchStartXRef.current = null;
  touchStartYRef.current = null;
  if (startX == null || startY == null) return;
  const t = e.changedTouches[0];
  if (!t) return;
  const dx = t.clientX - startX;
  const dy = t.clientY - startY;
  if (Math.abs(dx) < SWIPE_THRESHOLD_PX) return;
  if (Math.abs(dx) <= Math.abs(dy)) return;
  if (dx < 0) goPage(1);
  else goPage(-1);
};

// after
const onTouchStart = (e: ReactTouchEvent<HTMLDivElement>) => {
  const t = e.touches[0];
  if (!t) {
    touchStartXRef.current = null;
    touchStartYRef.current = null;
    return;
  }
  touchStartXRef.current = t.clientX;
  touchStartYRef.current = t.clientY;
};

const onTouchEnd = (e: ReactTouchEvent<HTMLDivElement>) => {
  const startX = touchStartXRef.current;
  const startY = touchStartYRef.current;
  touchStartXRef.current = null;
  touchStartYRef.current = null;
  if (startX == null || startY == null) return;
  if (isComposingRef.current) return; // IME 変換中は発火しない
  const t = e.changedTouches[0];
  if (!t) return;
  const dx = t.clientX - startX;
  const dy = t.clientY - startY;
  if (Math.abs(dx) < SWIPE_THRESHOLD_PX) return;
  if (Math.abs(dx) <= Math.abs(dy) * 2) return; // 水平優位 2:1
  if (dx < 0) goPage(1);
  else goPage(-1);
};
```

コメントブロックも B 案採用に更新する（既存の A 案 / B 案コメントを置き換え）。

## 受入条件
- 🔵 `isFromTextarea` 関数は削除
- 🔵 textarea 上でも |dx| >= 50 かつ |dx| > |dy|*2 のスワイプで navigate する
- 🔵 |dx| < 50 のスワイプは navigate しない
- 🔵 |dx|=30 / |dy|=60 のスワイプは navigate しない（縦優位）
- 🔵 IME 変換中のスワイプは navigate しない
- 🔵 既存 EditorPage.test.tsx の他テスト（日付挿入・オートページング・IME・50ページロック等）が全通過

## テスト方針
既存テスト `EditorPage.test.tsx` を更新:
- 「textarea 上のスワイプは navigate しない」系のテストを「textarea 上の水平スワイプで navigate する」に書き換え
- 新規追加:
  - 「textarea 上 |dx|=60 / |dy|=20 → navigate する」
  - 「textarea 上 |dx|=30 / |dy|=60 → navigate しない」
  - 「composition 中の textarea 上スワイプ → navigate しない」
- 既存「縦スクロール判定テスト」の座標が 2:1 仕様で通るか確認、必要なら調整

## 配線検証
- スワイプは既存 goPage 経由で flush → lastOpenedPage → navigate に流れる
- 新規関数は追加しない（既存 onTouchStart/onTouchEnd の修正のみ）

## 注意事項
- 既存の 50 ページ目ロック（onBeforeInput）、30 行自動ページング（checkOverflowAndNavigate）は変更なし
- transitionLockRef による多重発火抑止は既存のまま
