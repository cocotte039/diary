# M4-T6: 「書く」リンク削除 & 初回自動冊作成

## 目的
本棚と設定ページから「書く」リンクを削除し、本棚画面で冊が 0 件のときに `ensureActiveVolume` で自動的に 1 冊作成する。

## 変更対象
- ファイル:
  - `/home/cocotte/dev/diary/src/features/bookshelf/BookshelfPage.tsx`
  - `/home/cocotte/dev/diary/src/features/settings/SettingsPage.tsx`
- 変更内容:
  - `BookshelfPage`: ヘッダーから `<Link to="/">書く</Link>` を削除（残るのは「設定」リンク等）
  - `SettingsPage`: `<Link to="/">書く</Link>` を `<Link to="/">本棚</Link>` に置換（ラベルとリンク先の意味整合）
  - `BookshelfPage`: `useEffect` の volumes ロード後、`volumes.length === 0` なら `await ensureActiveVolume()` → 再ロード

## 依存する既存コード
- `/home/cocotte/dev/diary/src/lib/db.ts` `ensureActiveVolume()`（既存、active 0 → 新規作成）
- `/home/cocotte/dev/diary/src/features/bookshelf/BookshelfPage.tsx` 既存 `useEffect`
- `/home/cocotte/dev/diary/src/features/settings/SettingsPage.tsx` 既存ヘッダー

## 実装概要（3-5行）
1. `BookshelfPage.tsx` のヘッダー `<Link to="/">書く</Link>` を削除。
2. `useEffect` 内: `getAllVolumes()` の結果が空なら `ensureActiveVolume()` を呼び、再度 `getAllVolumes()` `getAllPages()` を取得して setState。
3. `SettingsPage.tsx` 内の同様リンクを「本棚」ラベル＋ `to="/"` に変更。
4. リンクテキスト `本棚` の表示位置・スタイルは既存に合わせる。

## 受入条件チェックリスト
- [ ] 🔵 BookshelfPage のヘッダーに「書く」リンクが存在しない
- [ ] 🔵 SettingsPage に「書く」リンクがなく、代わりに「本棚」リンクがある
- [ ] 🔵 冊 0 件状態で `BookshelfPage` をマウントすると 1 冊が作られ、空表示でなくカードが 1 枚表示される
- [ ] 🟡 自動作成は冪等（既に冊があれば追加されない） — 推測根拠: `ensureActiveVolume` の既存仕様

## Key Links（配線検証）
- `ensureActiveVolume` は `BookshelfPage.useEffect` から呼ばれる本番経路。
- 修正後の `<Link to="/">本棚</Link>` は `SettingsPage` の本番ヘッダーで描画される。

## テスト方針
- ユニット (Vitest + Testing Library + fake-indexeddb):
  - DB を完全空でセットアップ → `BookshelfPage` レンダ → 非同期完了後にカードが 1 枚表示される
  - 1 冊既存の状態 → 追加作成されないこと（volumes 数が 1 のまま）
  - `SettingsPage` レンダで「本棚」テキストかつ `href="#/"` のリンクが存在
- 目視: 初回起動 (DB クリア状態) で本棚に 1 冊が出る。

## 注意事項
- `ensureActiveVolume` 内で複数 active がある場合は最新以外を completed にする副作用がある。BookshelfPage の起動契機で常に呼ぶか、空件のときだけ呼ぶか要決定 → plan.md は「冊が0件なら呼ぶ」と明記なので 0 件時に限定する。
- 自動作成後の再ロードでチラつかないよう、setState のタイミングを `await` で揃える（`cancelled` フラグ確認は維持）。
