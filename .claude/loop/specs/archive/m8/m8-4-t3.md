# T8-4.3: BookshelfPage から deleteVolume 配線

## 変更対象
- `src/features/bookshelf/BookshelfPage.tsx`

## 依存
- T8-3.1（db.ts に deleteVolume 実装済み）
- T8-4.2（VolumeCard に onDelete prop 追加済み）

## 実装概要
BookshelfPage に `handleDelete` コールバックを追加し、VolumeCard に注入する。削除後は `reloadKey` を進めて本棚を再ロードする。

## 差分

```tsx
// import 追加
import {
  ...
  deleteVolume,
} from '../../lib/db';

// 関数追加
const handleDelete = useCallback(async (volumeId: string) => {
  try {
    await deleteVolume(volumeId);
  } catch (err) {
    // 静けさ原則: 失敗時もトーストは出さない。console だけ
    console.error('deleteVolume failed:', err);
  }
  setReloadKey((k) => k + 1);
}, []);

// VolumeCard に onDelete を渡す
<VolumeCard
  key={v.id}
  volume={v}
  pages={pagesByVolume.get(v.id) ?? []}
  initialPage={initialPages.get(v.id) ?? 1}
  onDelete={handleDelete}
/>
```

## 受入条件
- 🔵 `handleDelete` が `deleteVolume(id)` を呼び出し、setReloadKey する
- 🔵 VolumeCard に `onDelete` prop が渡される
- 🔵 削除後、該当 VolumeCard がグリッドから消える
- 🔵 削除後に active 冊が存在しない場合も、`ensureActiveVolume` の自動復旧で新しい空冊が 1 冊作られる（既存の useEffect 内 fallback）

## テスト方針
