# T8-4.2: VolumeCard 長押し削除配線

## 変更対象
- `src/features/bookshelf/VolumeCard.tsx`
- `src/features/bookshelf/BookshelfPage.module.css`（.card に `-webkit-touch-callout: none` 追加）

## 依存
- T8-4.1（LONG_PRESS_MS / LONG_PRESS_MOVE_TOLERANCE_PX が constants にあること）

## 実装概要
`VolumeCard` に `onDelete` prop を追加し、Pointer Events で 500ms 長押しを検知。長押し成立時に 2 段階 `window.confirm` を出し、両方 OK で `onDelete(volume.id)` を呼ぶ。ページ 0 枚の冊は 1 段階のみ。

## 実装方針

- Pointer Events で touch/mouse を一本化
- `onPointerDown`:
  - `pointerDownXRef = e.clientX; pointerDownYRef = e.clientY`
  - `longPressFiredRef = false`
  - `longPressTimerRef = setTimeout(() => { longPressFiredRef = true; handleLongPress(); }, LONG_PRESS_MS)`
- `onPointerMove`:
  - 開始座標から距離が `LONG_PRESS_MOVE_TOLERANCE_PX` を超えたら `clearTimeout(longPressTimerRef)`
- `onPointerUp` / `onPointerCancel` / `onPointerLeave`:
  - タイマー clear
- `onClick`（Link 上）:
  - `if (longPressFiredRef.current) { e.preventDefault(); e.stopPropagation(); longPressFiredRef.current = false; return; }`
  - 通常のクリック動作（Link 遷移）はそのまま

- `handleLongPress`:
  - pages.length 判定: 0 なら 1 段階、1 以上なら 2 段階 `window.confirm`
    - 1 段階目: `この冊を削除します。よろしいですか？`（0 ページ）or `この冊と全 ${pages.length} ページを削除します。よろしいですか？`（1 ページ以上）
    - 2 段階目（pages >= 1 時のみ）: `本当に削除しますか？この操作は取り消せません。`
  - 両方 OK なら `await onDelete(volume.id)`

## 受入条件
- 🔵 短いタップ（<500ms）で従来通り Link 遷移
- 🔵 500ms 以上の長押しで confirm が表示される（1 回目）
- 🔵 長押し中に LONG_PRESS_MOVE_TOLERANCE_PX を超えて指が動いたらキャンセル（confirm 出ない）
- 🔵 ページ 1 枚以上の冊は 2 段階 confirm、両方 OK で `onDelete` 呼び出し
- 🔵 ページ 0 枚の冊は 1 段階 confirm、OK で `onDelete` 呼び出し
- 🔵 confirm キャンセル時は `onDelete` 呼ばれない
- 🔵 長押し成立時の onClick は preventDefault され Link 遷移しない

- このタスクで配線完了後、`grep -rn "onDelete" src/features/bookshelf/` で VolumeCard.tsx と BookshelfPage.tsx の 2 箇所ヒットするはず（T8-4.3 で確定）

## 注意事項
- Pointer Events の polyfill は現代ブラウザで不要
- React の SyntheticPointerEvent は type 付きで扱う: `PointerEvent<HTMLDivElement>`
- `onContextMenu={(e) => e.preventDefault()}` も併用すると mouse 右クリックの誤発火防止に有効（任意）
- 既存の `Link to={`/book/${volume.id}/${initialPage}`}>...</Link>` の構造は保持
