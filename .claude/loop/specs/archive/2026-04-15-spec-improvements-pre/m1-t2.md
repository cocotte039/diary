# M1-T2: pagination.ts 関数書き換え

## 変更対象
- `src/lib/pagination.ts`

## 実装概要
1. `splitAtLine30` を `splitAtCharLimit` にリネーム。実装:
   ```ts
   export function splitAtCharLimit(text: string): { keep: string; overflow: string } {
     if (text.length <= CHARS_PER_PAGE) return { keep: text, overflow: '' };
     return { keep: text.slice(0, CHARS_PER_PAGE), overflow: text.slice(CHARS_PER_PAGE) };
   }
   ```
2. `countPages(text)` を `Math.max(1, Math.ceil(text.length / CHARS_PER_PAGE))` に
3. `getPageNumber(selectionStart, text)` は `Math.floor(clamped / CHARS_PER_PAGE) + 1` 相当に（clamped = min(selectionStart, text.length)）
4. `splitIntoPages(text)` / `joinPages(pages)` は **呼び出し箇所を grep 確認**。使われていれば文字数ベースで書き換え、未使用なら削除して JSDoc に「未使用のため削除」明記
5. `countLogicalLines` は罫線用にそのまま残す
6. `getScrollTopForCursor(text, selectionStart)` はロジック維持（行インデックス * LINE_HEIGHT_PX）。呼び出し側（M3 で useEditorCursor を見る）でスクロールコンテナが変わるが、y 計算は同じ

## 受入条件
- 🔵 `splitAtCharLimit('')` → `{ keep: '', overflow: '' }`
- 🔵 `splitAtCharLimit('a'.repeat(1200))` → `{ keep: 'a'.repeat(1200), overflow: '' }`
- 🔵 `splitAtCharLimit('a'.repeat(1201))` → `{ keep: 'a'.repeat(1200), overflow: 'a' }`
- 🔵 `countPages('a'.repeat(2401)) === 3`
- 🔵 `countPages('')===1`
- 🟡 未使用関数の削除判断は grep 結果に従う

## grep 必須
- `splitAtLine30` / `splitIntoPages` / `joinPages` / `getPageNumber` / `getScrollTopForCursor` の呼び出し箇所をすべて確認
