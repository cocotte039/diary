# M4-T2: Volume 型拡張と DB v2 マイグレーション

## 目的
ページ単位保存と「最後に開いたページ」記憶を可能にするため、`Volume` 型を拡張し DB を v2 に上げる。`savePage` / `updateVolumeLastOpenedPage` / `getLatestUpdatedPageNumber` を追加。

## 変更対象
- ファイル:
  - `/home/cocotte/dev/diary/src/types/index.ts`
  - `/home/cocotte/dev/diary/src/lib/constants.ts`
  - `/home/cocotte/dev/diary/src/lib/db.ts`
- 関数（新規）:
  - `savePage(volumeId: string, pageNumber: number, content: string): Promise<Page>`
  - `updateVolumeLastOpenedPage(volumeId: string, pageNumber: number): Promise<void>`
  - `getLatestUpdatedPageNumber(volumeId: string): Promise<number>`（更新ページが無ければ 1 を返す）
- 型変更:
  - `Volume.lastOpenedPage?: number`（optional 1〜50）
- 定数変更:
  - `DB_VERSION = 2`

## 依存する既存コード
- `/home/cocotte/dev/diary/src/lib/db.ts` の `getDB()` `upgrade(db, oldVersion, newVersion, transaction)`（idb の openDB シグネチャ）
- 既存 `Page.updatedAt` (ISO8601)
- 既存 `getPage(volumeId, pageNumber)` でのインデックス `by-volume-page`
- 既存 `nowIso()`, `uuid()`

## 実装概要（3-5行）
1. `types/index.ts` に `lastOpenedPage?: number` を追加。
2. `constants.ts` で `DB_VERSION = 2` に変更。
3. `db.ts` の `openDB` の `upgrade` コールバックを `(db, oldVersion)` シグネチャに拡張し、`if (oldVersion < 2) {}`（optional フィールド追加のみのため空でよい）を入れる。既存 v1 ストア作成は `if (oldVersion < 1)` でガード。
4. `savePage`: 単一トランザクション (`pages` `readwrite`) で `getFromIndex('by-volume-page', [volumeId, pageNumber])` → 存在すれば content/updatedAt/syncStatus=pending 更新、無ければ新規 Page を put。**他ページに一切触れない**。
5. `updateVolumeLastOpenedPage`: `volumes` ストアで `get` → `lastOpenedPage = pageNumber` → `put`。
6. `getLatestUpdatedPageNumber`: `getPagesByVolume(volumeId)` → updatedAt 降順で先頭の `pageNumber`、空なら 1。

## 受入条件チェックリスト
- [ ] 🔵 `DB_VERSION === 2`
- [ ] 🔵 `Volume.lastOpenedPage` が optional number
- [ ] 🔵 `savePage(id, n, text)` が単一ページのみを更新し、他のページの updatedAt を変えない
- [ ] 🔵 v1 で作られた既存データが v2 オープン後も全件読める
- [ ] 🟡 `getLatestUpdatedPageNumber` は updatedAt 最新ページを返す（空冊なら 1） — 推測根拠: plan.md C4 フォールバック仕様

## テスト方針
- ユニット (Vitest + fake-indexeddb):
  - v1 スキーマで volumes/pages を仕込み → DB を閉じて再 open（v2）→ `getAllVolumes` `getPagesByVolume` で全件取得できる
  - `savePage(id, 3, 'foo')` を呼ぶ前後で、ページ 1,2,4,5 の `updatedAt` が変化しないこと
  - 新規ページ作成: 存在しない pageNumber に savePage → 新 Page が作成され `id` が UUID
  - `updateVolumeLastOpenedPage` 後に `getVolume` で `lastOpenedPage` が取れる
  - `getLatestUpdatedPageNumber`: ページが無い空冊で 1、ある場合は最新更新のページ番号
- TDD 推奨: テストを先に書き、現状 0 件 → savePage 実装で Green。

## 注意事項
- 既存 `saveVolumeText` は M7 まで残るが、新コードからは呼ばない（C1 他ページ破壊回避）。
